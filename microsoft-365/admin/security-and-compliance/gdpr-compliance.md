## <a name="a-quick-overview-of-the-gdpr"></a><span data-ttu-id="05263-114">GDPR에 대한 간략한 개요</span><span class="sxs-lookup"><span data-stu-id="05263-114">A quick overview of the GDPR</span></span>

<span data-ttu-id="05263-120">GDPR의 주요 내용은 데이터와 사용 방법에 관한 것입니다.</span><span class="sxs-lookup"><span data-stu-id="05263-120">The GDPR is all about data and how it's used.</span></span> <span data-ttu-id="05263-121">데이터에 수명 주기가 있다고 생각하세요.</span><span class="sxs-lookup"><span data-stu-id="05263-121">Think of data as having a life cycle.</span></span> <span data-ttu-id="05263-122">이 주기는 데이터를 수집할 때 시작되고, 데이터를 저장하고 사용(처리)할 때 계속되며, 시스템에서 완전히 삭제하면 종료됩니다.</span><span class="sxs-lookup"><span data-stu-id="05263-122">The cycle starts when you collect data, continues as you store it and use it (processing), and ends when you completely delete it from your systems.</span></span>

- <span data-ttu-id="05263-124">**개인 데이터:** 데이터를 개인에 연결하고 식별할 수 있는 경우 해당 데이터는 GDPR과 관련하여 개인 데이터로 간주됩니다.</span><span class="sxs-lookup"><span data-stu-id="05263-124">**Personal data:** If you can link data to an individual and identify them, then that data is considered personal with respect to the GDPR.</span></span> <span data-ttu-id="05263-125">개인 데이터의 예로는 이름, 주소, 생년월일 및 IP 주소가 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-125">Examples of personal data include name, address, date of birth, and IP address.</span></span> <span data-ttu-id="05263-126">GDPR은 데이터를 개인에 연결할 수 있는 경우 데이터가 얼마나 모호하거나 기술적인지에 관계없이 인코딩된 정보("가명 정보"라고도 함)도 개인 데이터로 간주합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-126">The GDPR considers even encoded information (also known as "pseudonymous" information) to be personal data, regardless of how obscure or technical the data is, if the data can be linked to an individual.</span></span>

## <a name="gdpr-terms"></a><span data-ttu-id="05263-131">GDPR 용어</span><span class="sxs-lookup"><span data-stu-id="05263-131">GDPR terms</span></span>

<span data-ttu-id="05263-132">다음에는 GDPR에서 자주 참조되는 몇 가지 용어가 표시됩니다.</span><span class="sxs-lookup"><span data-stu-id="05263-132">You'll see some terms referred to frequently in the GDPR.</span></span> <span data-ttu-id="05263-133">이러한 용어를 이해하는 것이 중요합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-133">It's important to understand these terms.</span></span>
  
 <span data-ttu-id="05263-134">**동의**</span><span class="sxs-lookup"><span data-stu-id="05263-134">**Consent**</span></span>
  
<span data-ttu-id="05263-135">GDPR은 "개인 데이터의 처리는 인간에게 서비스를 제공하도록 설계되어야 ㄷ한다"라고 명시합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-135">The GDPR states: "The processing of personal data should be designed to serve mankind."</span></span> <span data-ttu-id="05263-136">GDPR은 개인 데이터를 처리할 때 동의를를 통해 이러한 목표를 달성하려고 합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-136">The GDPR hopes to achieve this goal by using consent when processing personal data.</span></span> <span data-ttu-id="05263-137">해당 작업은 고객에게 회사로부터 이메일 메시지를 받을 것인지 묻는 간단한 작업일 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-137">That could be the simple act of asking your customers if they want to receive email messages from your company.</span></span> <span data-ttu-id="05263-138">또한 마케팅에 데이터를 사용하려는 경우 웹 사이트에 더 이상 옵트아웃 확인란이 없음을 의미합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-138">It also means no more opt-out check boxes on your website when you want to use data for marketing.</span></span> <span data-ttu-id="05263-139">"명확한 확인 행위"를 사용하여 명시적인 동의를 얻어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-139">You must take explicit consent using a "clear affirmative act".</span></span> <span data-ttu-id="05263-140">또한 동의를 취하거나 해지할 때에 관한 기록도 유지해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-140">And, you will need to also keep records of when a consent is taken or revoked.</span></span>
  
 <span data-ttu-id="05263-141">**데이터 주체 권리**</span><span class="sxs-lookup"><span data-stu-id="05263-141">**Data subject rights**</span></span>
  
<span data-ttu-id="05263-142">GDPR은 데이터 주체 권한을 설정합니다. 즉, 고객, 직원, 비즈니스 파트너, 클라이언트, 계약자, 학생, 공급자 등은 그들의 개인 데이터에 대해 다음을 수행할 수 있는 권한이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-142">The GDPR establishes data subject rights, which means that, with respect to their personal data, customers, employees, business partners, clients, contractors, students, suppliers, and so forth have the right to:</span></span>
  
- <span data-ttu-id="05263-143">**데이터에 대한 알림:** 개인에게 데이터 사용에 대해 알려야 합니다.</span><span class="sxs-lookup"><span data-stu-id="05263-143">**Be informed about their data:** You must inform individuals about your use of their data.</span></span> 
    
- <span data-ttu-id="05263-144">**데이터에 대한 액세스 권한 보유:** 사용자가 보유한 데이터에 대한 액세스 권한을 개인에게 부여해야 합니다(예: 계정 액세스를 사용하거나 수동 방식으로).</span><span class="sxs-lookup"><span data-stu-id="05263-144">**Have access to their data:** You must give individuals access to any of their data that you hold (for example, by using account access or in some manual manner).</span></span> 
    
- <span data-ttu-id="05263-145">**데이터 수정 요청:** 개인은 부정확한 데이터를 수정하도록 요청할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-145">**Ask for data rectification:** Individuals can ask you to correct inaccurate data.</span></span> 
    
- <span data-ttu-id="05263-146">**삭제할 데이터 요청:** '삭제권'이라고도 하는 이 권리를 통해 개인은 회사에서 수집한 개인 데이터를 사용하거나 공유하는 모든 시스템에서 삭제하도록 요청할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-146">**Ask for data to be deleted:** Also known as the 'right to erasure', this right allows an individual to request that any of their personal data a company has collected is deleted across all systems that use it or share it.</span></span> 
    
- <span data-ttu-id="05263-147">**제한된 처리 요청:** 개인은 데이터를 표시하지 않거나 제한하도록 요청할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-147">**Request restricted processing:** An individual can ask that you suppress or restrict their data.</span></span> <span data-ttu-id="05263-148">그러나 이 권리는 특정 상황에서만 적용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-148">However, it is only applicable under certain circumstances.</span></span> 
    
- <span data-ttu-id="05263-149">**데이터 이동성 보유:** 개인이 데이터를 다른 회사로 전송하도록 요청할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-149">**Have data portability:** An individual can ask for their data to be transferred to another company.</span></span> 
    
- <span data-ttu-id="05263-150">**개체:** 개인은 직접 마케팅을 포함하여 다양한 용도로 사용되는 데이터에 개체를 지정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-150">**Object:** An individual can object to their data being used for various uses including direct marketing.</span></span> 
    
- <span data-ttu-id="05263-151">**프로파일링을 포함하여 자동화된 의사 결정의 적용을 받지 않도록 요청:** GDPR에는 데이터를 사용하여 사용자를 프로파일링하고 해당 프로파일링에 따라 의사 결정을 자동화하는 행위에 대한 엄격한 규칙이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="05263-151">**Ask not to be subject to automated decision-making, including profiling:** The GDPR has strict rules about using data to profile people and automate decisions based on that profiling.</span></span>
