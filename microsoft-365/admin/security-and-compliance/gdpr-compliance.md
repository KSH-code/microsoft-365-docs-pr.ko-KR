# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR 간소화: 중소기업을 위한 가이드

 *비즈니스용 Microsoft 365를 사용하여 GDPR 규정 준수 완화 및 관리* 
  
GDPR(일반 데이터 보호 규정)은 조직이 개인 데이터를 처리하는 방법을 규정하는 유럽 연합(EU) 규정입니다. GDPR은 2018년 5월 25일 시행될 예정입니다. 비즈니스가 유럽 연합의 시민에게 서비스를 판매하거나 서비스를 제공하거나 고용하는 경우 [GDPR은](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) 사용자에게 영향을 미치게 됩니다. 
  
이 문서는 GDPR이 무엇이고, GDPR이 어떤 이유인지, 비즈니스용 Microsoft 365가 조직이 GDPR을 준수하는 데 어떤 도움을 줄 수 있도록 하는지 이해하는 데 도움이 됩니다.
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR에 대한 간략한 개요

GDPR은 1995년에 처음 제정된 이전 DPD(데이터 보호 지시문)를 업데이트하고 확장하는 EU 규정입니다. GDPR은 개별 클라이언트, 고객, 직원 또는 비즈니스 파트너인 개인 데이터의 개인 정보 보호와 관련되어 있습니다. GDPR의 목표는 EU 또는 다른 곳에 있는 EU 시민을 위한 개인 데이터 보호를 강화하는 것입니다. 규정은 기대치를 설정하고 이를 달성하는 방법에 대해 권고합니다. 2018년 5월 말에 GDPR이 적용 가능해지는 경우 조직은 GDPR의 요구 사항을 충족하는 조치를 마련해야 합니다.
  
GDPR은 데이터와 데이터 사용 방법에 관한 것입니다. 데이터를 수명 주기가 있는 것으로 생각하면 됩니다. 주기는 데이터를 수집할 때 시작되고 데이터를 저장하고 사용할 때 계속 사용(처리)하며 시스템에서 완전히 삭제하면 종료됩니다. 
  
GDPR은 다음과 같은 유형의 데이터와 관련이 있습니다. 
  
- **개인 데이터:** 개인에 데이터를 연결하고 식별할 수 있는 경우 해당 데이터는 GDPR과 관련한 개인 데이터로 간주됩니다. 개인 데이터의 예로는 이름, 주소, 생년월일 및 IP 주소가 있습니다. GDPR은 데이터가 개인에게 연결될 수 있는 경우 데이터가 얼마나 가명되거나 기술적이지와 관계없이 인코딩된 정보("가명" 정보라고도 알려지기)를 개인 데이터로 고려합니다.
    
- **중요한 개인 데이터** 개인 데이터에 세부 정보를 추가하는 데이터입니다. 예로는 근로자, 노조 회원, 교신 출처 등입니다. 중요한 개인 데이터에는 생체 인식 데이터와도 포함됩니다. GDPR에서 중요한 데이터에는 개인 데이터보다 더 엄격한 보호 규칙이 있습니다. 
    
## <a name="gdpr-terms"></a>GDPR 약관

GDPR에서 자주 언급되는 몇 가지 용어를 볼 수 있습니다. 이러한 용어를 이해하는 것이 중요합니다.
  
 **동의**
  
GDPR은 다음을 규정합니다. "개인 데이터 처리는 인간을 위해 설계해야 합니다." GDPR은 개인 데이터를 처리 할 때 동의를 사용하여 이 목표를 달성하기를 기대합니다. 이는 고객에게 회사로부터 전자 메일 메시지를 받을지 묻는 간단한 행동일 수 있습니다. 또한 마케팅에 데이터를 사용하려는 경우 웹 사이트에서 더 이상 옵트아웃 확인란을 사용하지 않습니다. "명확한 객관적 행위"를 사용하여 명시적 동의를 취해야 합니다. 또한 동의가 취해지거나 해지된 경우 기록을 보관해야 합니다.
  
 **데이터 주체 권한**
  
GDPR은 데이터 주체 권한을 수립합니다. 즉, 개인 데이터, 고객, 직원, 비즈니스 파트너, 클라이언트, 계약자, 학생, 공급 업체 등은 다음을 할 수 있는 권한을 확정합니다.
  
- **데이터에 대한 정보를 제공해야 합니다.** 개인에게 데이터 사용에 대해 알려야 합니다. 
    
- **데이터에 액세스할 수 있습니다.** 개인에게 보유한 모든 데이터에 대한 액세스 권한을 부여해야 합니다(예: 계정 액세스 사용 또는 수동 방식 사용). 
    
- **데이터 정정을 요청합니다.** 개인이 부정확한 데이터를 수정해달고 요청할 수 있습니다. 
    
- **삭제할 데이터를 요청합니다.** '삭제권'라고도 하는 이 권리는 개인이 회사에서 수집한 개인 데이터가 해당 데이터를 사용하거나 공유하는 모든 시스템에서 삭제될 것을 요청할 수 있도록 허용합니다. 
    
- **제한된 처리 요청:** 개인이 데이터를 표시하지 못하도록 요구하거나 제한할 수 있습니다. 그러나 특정 상황에서만 적용할 수 있습니다. 
    
- **데이터 이식성이 있습니다.** 개인은 데이터를 다른 회사로 전송할 수 있도록 요구할 수 있습니다. 
    
- **개체:** 개인은 직접 마케팅을 비롯한 다양한 용도에 사용되는 데이터에 대해 이의를 할 수 있습니다. 
    
- 프로파일링을 포함하여 자동화된 의사 결정의 적용을 **묻지 말아야 합니다.** GDPR에는 데이터를 사용하여 사용자 프로필을 작성하고 프로파일링에 따라 의사 결정을 자동화하는 엄격한 규칙이 있습니다. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>GDPR에 대한 일반적인 중소기업 질문

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>전적으로 소유주입니다. GDPR에 대해 걱정해야 하나요?

GDPR은 처리하는 데이터에 대한 정보로, 직원 수가 아니라 처리합니다. 단독 소유주라도 모든 크기의 회사에 영향을 미치게 됩니다. 그러나 직원 수가 250명 미만인 회사는 레코드 유지 감소와 같은 일부 면제가 있지만, 데이터 처리가 개인의 권리에 영향을 주지 않는지 확실한 경우와 가끔씩 처리해야 합니다.
  
예를 들어 비개인 데이터 처리는 면제되거나 축소된 조치가 필요합니다. 그러나 "특수 범주 중요한 데이터"로 보인 데이터를 처리하는 경우 간혹 이 데이터 처리를 기록해야 합니다. "가끔씩 처리"라는 정의는 모호하지만 한 번 또는 드물게 사용되는 데이터에 적용하기 위한 것입니다.
  
또한 수집하는 개인 데이터가 보호되어 있는지도 확인해야 합니다. 즉, 암호화하고 적어도 암호를 사용하여 액세스 권한을 제어해야 합니다. 고객 데이터를 보호하지 않은 데스크톱의 스프레드시트에 유지하면 GDPR 기대치가 충족되지 않습니다. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>회사 웹 사이트가 GDPR을 준수하는지 어떻게 알 수 있나요?

가장 먼저 묻는 질문은 사이트의 어디에서나 개인 데이터를 수집하나요? 예를 들어 이름과 전자 메일 주소를 요청하는 연락처 양식이 있을 수 있습니다. 마케팅 전자 메일을 보내고 싶은 경우 데이터를 사용할 정확한 정보를 설명하는 '옵트인' 확인란을 추가해야 합니다. 받는 사람이 해당 확인란을 선택해야만 마케팅 목적으로 개인 데이터를 사용할 수 있습니다.
  
또한 데이터를 저장하는 데이터베이스가 보호되어 있는지도 확인할 수 있습니다. 웹 호스팅 회사 또는 클라우드 저장소 공급업체에서 이에 대해 조언할 수 있습니다. 비즈니스용 Microsoft 365를 사용하는 경우 데이터 저장은 GDPR을 준수합니다. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>회사가 유럽 외부에 있습니다. GDPR이 실제로 영향을 미치나요?

GDPR은 EU 시민을 보호하는 규정입니다. 회사에서 지금 EU 시민을 상대하거나 앞으로도 영향을 받을 것으로 기대하는 경우 이는 EU 주에 거주하는 시민과 다른 곳에 거주하는 시민 모두에 적용됩니다.
  
다음 예제를 고려해 보십시오.
  
- EU 시민에게 자동차를 고용하는 미국 회사에서는 고객의 데이터를 수집하고 처리할 때 GDPR 요구 사항을 충족해야 합니다. 회사는 고객의 데이터를 사용할 때 동의해야 합니다. 데이터가 안전하게 저장되도록 보장해야 합니다. 또한 고객이 모든 데이터 주체 권한을 적용할 수 있는지도 확인해야 합니다. 
    
- 오스트레일리아 회사에서 온라인으로 제품을 판매하고 해당 사용자는 온라인 계정을 설정했습니다. GDPR 데이터 주체 권리 및 동의는 계정을 여는 EU 시민에게 적용됩니다. 회사는 고객이 모든 데이터 주체 권한을 적용할 수 있는지 확인해야 합니다. 
    
- 국제 자선 단체는 후원자에 대한 데이터를 수집하고 이를 사용하여 업데이트 및 후원 요청을 보낼 수 있습니다. GDPR 상태: '... 직접 마케팅 목적으로 개인 데이터 처리는 정당한 이익을 위해 수행된 것으로 생각될 수 있습니다." 그러나 데이터 주체의 관심사에 대해 입증할 책임은 조직에 있습니다. 회사(또는 이 경우 자선 조직)는 항상 명시적, 명시적 동의를 얻어야 합니다.
    
GDPR은 고객 데이터가 테두리를 넘어 이동하는 경우도 적용됩니다. 데이터 저장소에 클라우드 컴퓨팅을 사용하는 경우 서비스가 GDPR을 완전히 준수하는지 확인해야 합니다. 데이터 저장소가 데이터 보호의 불량 레코드가 있는 위치에 있는 경우 복잡할 수 있습니다. 비즈니스용 Microsoft 365를 사용하는 경우 GDPR 요구 사항을 충족하기 위한 올바른 법적 문서가 준비됩니다.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>물론 데이터를 수집하지만 다른 회사에서는 데이터를 저장합니다. 이 경우 후크를 벗어날 수 있나요?

GDPR에서 데이터를 수집하는 경우 어느 정도 영향을 받을 수 있습니다. GDPR에는 데이터 프로세서와 데이터 컨트롤러의 개념이 있습니다. 
  
- **데이터 컨트롤러:** 데이터 수집 방법, 무엇 및 이유를 결정하는 개인 또는 조직(조인트 컨트롤러가 있는 경우)입니다. 다른 회사의 클라우드 서버를 사용하여 저장할 수 있습니다. 예를 들어 고객 데이터를 수집하는 웹 사이트는 컨트롤러입니다. 
    
- **데이터 프로세서:** 컨트롤러를 대신하여 데이터를 저장하고 요청 시 이러한 데이터를 처리하는 개인 또는 조직입니다. 예를 들어 비즈니스용 Microsoft 365 앱 데이터 저장소는 프로세서 역할을 하며 완전히 GDPR을 준수합니다. 
    
    조직 또는 시스템은 컨트롤러와 프로세서 역할을 할 수 있습니다. 비즈니스용 Microsoft 365는 GDPR과 GDPR을 모두 준수하는 역할을 할 수 있습니다.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>이전 고객에게 마케팅 전자 메일을 보낼 수 있나요?

몇 년 동안 했던 고객도 마케팅에 데이터를 사용하기로 동의하는지 확인해야 합니다. 이전에 동의를 캡처한 것일 수 있을 뿐만 아니라 이를 표시하는 레코드도 있습니다. 그렇다면 마케팅을 계속할 수 있습니다. 그렇지 않은 경우 고객에게 마케팅을 계속하기 위한 권한을 고객에게 제공해야 합니다. 일반적으로 고객에게 사이트로 이동하여 향후 전자 메일을 받기 위한 동의 옵션을 선택하도록 요청하는 전자 메일을 보내는 과정입니다. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>새 직원을 모집할 때 GDPR에 대해 걱정해야 하나요? 현재 직원은 어떻게 하나요?

GDPR은 고객 데이터에만 영향을 주지 않습니다. 직원 데이터로도 확장됩니다. 신규 채용은 종종 LinkedIn과 같은 소셜 미디어 플랫폼을 사용하여 있습니다. 가능한 모집 데이터를 특정 개인의 익스프레스 권한 없이 저장하지 않는지 확인
  
기존 직원과 신입 직원 계약의 경우 계약 종료 시 서명이 반드시 동의할 필요는 없습니다. 특히 계약에 객관적이지 않은 조항이 사용되는 경우입니다. 이 경우 절과 연결된 명시적인 방식으로 동의를 캡처해야 합니다. 이는 직원 계약에 따라 달라지지만 경우에 따라 "적법한 이익"을 사용하여 직원 데이터 처리 통지를 추가하여 직원이 해당 데이터로 할 일을 인식할 수 있도록 할 수 있습니다.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>비즈니스용 Microsoft 365를 사용하여 개인 정보 보호 문제 충족

GDPR을 준수하는 것은 개인 데이터가 보호되어 있는지를 보장하는 것입니다. GDPR에는 디자인 및 기본으로 개인 정보 보호 개념이 있습니다. 즉, 개인 정보 보호를 충족하는 것이 두 번째 특성이 될 수 있도록 시스템 및 제품에 데이터 보호를 "제과"해야 합니다. 
  
대기업과 마찬가지로 중소기업은 보안을 위해 편의를 제공해야 합니다. 비즈니스용 Microsoft 365는 직원 수가 300명 미만인 회사를 위해 고안된 것입니다. 중소기업은 Microsoft 클라우드 기반 도구를 사용하여 비즈니스 생산성을 향상시킬 수 있습니다. 비즈니스용 Microsoft 365를 통해 중소기업은 전자 메일, 설명서 및 모임 및 이벤트를 관리할 수 있습니다. 또한 GDPR 규정 준수에 필수적인 기본 제공 보안 조치 및 장치 관리도 제공합니다.
  
비즈니스용 Microsoft 365는 다음과 같은 방식으로 GDPR 프로세스를 지원할 수 있습니다.
  
- **검색:** GDPR 규정 준수에 대한 중요한 단계는 현재 데이터를 아는 것입니다. 
    
- **관리:** 데이터에 대한 액세스를 제어하고 데이터 사용을 관리하는 것은 GDPR의 중요한 부분입니다. 비즈니스용 Microsoft 365는 장치에 적용하려는 정책에 따라 비즈니스 데이터를 보호합니다. 디바이스 관리는 직원이 원격으로 작업하는 연령대에서 중요합니다. 비즈니스용 Microsoft 365에는 모든 장치에서 데이터를 보호하는 장치 관리 기능이 포함되어 있습니다. 예를 들어 비즈니스에 있는 모든 Windows 10 디바이스가 2016년 10월을 통해 보호된다고 지정할 수 Windows Defender. 
    
- **보호:** 비즈니스용 Microsoft 365는 보안을 위해 고안된 것입니다. 장치 관리 및 데이터 보호 컨트롤은 원격 장치를 포함하여 비즈니스 네트워크 전체에서 작동하여 데이터를 안전하게 보호합니다. 비즈니스용 Microsoft 365는 Office 응용 프로그램의 개인 정보 설정 및 문서 암호화와 같은 컨트롤을 제공합니다. 비즈니스용 Microsoft 365를 사용하면 GDPR 규정 준수 모니터링을 수행하여 올바른 수준의 보호를 설정해야 합니다. 
    
- **보고서:** GDPR은 보고에 많은 강조를 합니다. 직원이 한 명인 기업에서도 많은 양의 데이터를 처리하면 절차를 문서화하고 보고해야 합니다. 비즈니스용 Microsoft 365는 소규모 조직에 대한 보고 요구 사항을 충족하지 않습니다. 
    
    감사 로그와 같은 도구를 사용하면 데이터 이동을 추적하고 보고할 수 있습니다. 보고서에는 수집 및 저장하는 데이터, 데이터로 무엇을 하는지, 데이터 전송을 분류하는 것이 포함됩니다. 
    
고객, 직원 및 클라이언트는 데이터 개인 정보 보호의 중요성을 더 잘 인식하고 있으며 이제는 회사나 조직이 해당 개인 정보를 준수할 것으로 기대합니다. 비즈니스용 Microsoft 365는 비즈니스에 큰 업사이브 없이 GDPR 규정 준수를 달성하고 유지 관리하는 도구를 제공합니다.
