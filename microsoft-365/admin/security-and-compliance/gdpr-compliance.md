# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR 간소화: 중소기업을 위한 가이드

 *비즈니스용 Microsoft 365를 사용하여 GDPR 규정 준수 완화 및 관리* 
  
GDPR(일반 데이터 보호 규정)은 조직이 개인 데이터를 처리하는 방법을 규정하는 유럽 연합(EU) 규정입니다. GDPR은 2018년 5월 25일에 시행될 예정입니다. 비즈니스가 유럽 연합에 서비스를 판매하거나, 서비스를 제공하거나, 유럽 연합 시민을 고용하는 경우 [GDPR은 사용자에게](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) 영향을 미치게 됩니다. 
  
이 문서는 GDPR의 내용, 그 이유 및 비즈니스용 Microsoft 365가 조직이 GDPR을 준수하는 데 어떤 도움을 줄 수 있도록 하는지 이해하는 데 도움이 됩니다.
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR에 대한 간략한 개요

GDPR은 1995년에 처음 제정된 이전 DPD(데이터 보호 지시문)를 업데이트하고 확장하는 EU 규정입니다. GDPR은 개인 데이터의 개인 정보 보호, 즉 개별 클라이언트, 고객, 직원 또는 비즈니스 파트너와 관련되어 있습니다. GDPR의 목표는 EU 또는 다른 곳에 있는 EU 시민에 대한 개인 데이터 보호를 강화하는 것입니다. 규정은 기대치를 설정하고 이를 달성하는 방법에 대해 권고합니다. 2018년 5월 말에 GDPR이 적용될 때 조직은 GDPR의 요구 사항을 충족하는 조치를 마련해야 합니다.
  
GDPR은 데이터 및 데이터 사용 방법에 관한 것입니다. 데이터를 수명 주기가 있는 것으로 생각하면 됩니다. 주기는 데이터를 수집할 때 시작되고 데이터를 저장하고 사용할 때 계속 사용(처리)하며 시스템에서 완전히 삭제하면 종료됩니다. 
  
GDPR은 다음과 같은 유형의 데이터와 관련이 있습니다. 
  
- **개인 데이터:** 개인에 데이터를 연결하고 식별할 수 있는 경우 해당 데이터는 GDPR과 관련한 개인 데이터로 간주됩니다. 개인 데이터의 예로는 이름, 주소, 생년월일 및 IP 주소가 있습니다. GDPR은 인코딩된 정보("가명" 정보라고도 불림)는 데이터가 개인에 연결될 수 있는 경우 데이터의 가명 또는 기술에 관계없이 개인 데이터로 고려합니다.
    
- **중요한 개인 데이터** 개인 데이터에 세부 정보를 추가하는 데이터입니다. 예로는 근로자, 노동 조합 회원, 인종 근원 등이 있습니다. 중요한 개인 데이터에는 생체 인식 데이터와도 포함됩니다. GDPR에서 중요한 데이터에는 개인 데이터보다 더 엄격한 보호 규칙이 있습니다. 
    
## <a name="gdpr-terms"></a>GDPR 약관

GDPR에서 자주 언급되는 몇 가지 용어가 볼 수 있습니다. 이러한 용어를 이해하는 것이 중요합니다.
  
 **동의**
  
GDPR은 "개인 데이터 처리는 인간을 위해 설계해야 합니다." GDPR은 개인 데이터를 처리 할 때 동의를 사용하여 이 목표를 달성하기를 기대합니다. 이는 고객에게 회사에서 보낸 전자 메일 메시지를 받을지 묻는 간단한 행위일 수 있습니다. 또한 마케팅에 데이터를 사용하려는 경우 웹 사이트에서 더 이상 옵트아웃 확인란이 없음을 의미하기도 합니다. "명확한 객관적 행위"를 사용하여 명시적 동의를 취해야 합니다. 또한 동의가 취해지거나 해지된 경우 기록도 보관해야 합니다.
  
 **데이터 주체 권리**
  
GDPR은 데이터 주체 권한을 수립합니다. 즉, 개인 데이터, 고객, 직원, 비즈니스 파트너, 클라이언트, 계약자, 학생, 공급 업체 등은 다음을 할 수 있는 권한을 확립합니다.
  
- **데이터에 대한 정보를 제공해야 합니다.** 개인에게 데이터 사용에 대해 알려야 합니다. 
    
- **데이터에 대한 액세스 권한이 있습니다.** 개인에게 보유한 모든 데이터에 대한 액세스 권한을 부여해야 합니다(예: 계정 액세스 사용 또는 수동 방식 사용). 
    
- **데이터 정정을 요청합니다.** 개인은 부정확한 데이터를 수정해달라 요청할 수 있습니다. 
    
- **삭제할 데이터를 요청합니다.** '삭제권'으로도 알려진 이 권리는 개인이 회사에서 수집한 개인 데이터가 해당 데이터를 사용하거나 공유하는 모든 시스템에서 삭제될 것을 요청할 수 있도록 허용합니다. 
    
- **제한된 처리 요청:** 개인은 데이터를 표시하지 또는 제한하도록 요구할 수 있습니다. 그러나 특정 상황에서만 적용할 수 있습니다. 
    
- **데이터 이식성이 있습니다.** 개인은 데이터를 다른 회사로 전송할 수 있도록 요구할 수 있습니다. 
    
- **개체:** 개인은 직접 마케팅을 비롯한 다양한 용도에 사용되는 데이터에 대해 이의를 할 수 있습니다. 
    
- 프로파일링을 포함하여 자동화된 의사 결정의 적용을 **묻지 않습니다.** GDPR에는 데이터를 사용하여 사용자 프로필을 작성하고 프로파일링에 따라 의사 결정을 자동화하는 엄격한 규칙이 있습니다. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>GDPR에 대한 일반적인 중소기업 질문

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>전적으로 소유주입니다. GDPR에 대해 걱정해야 하나요?

GDPR은 직원이 있는 수가 아니라 처리되는 데이터에 대한 것입니다. 모든 기업에 영향을 미치며 전적으로 소유주에게도 영향을 미치게 됩니다. 그러나 직원 수가 250명 미만인 회사는 레코드 유지 감소와 같은 일부 면제가 있지만, 데이터 처리가 개인의 권리에 영향을 주지 않는지 그리고 가끔씩 처리해야 하는지 확실한 경우입니다.
  
예를 들어, 비개인 데이터 처리는 면제되거나 축소된 조치가 필요합니다. 그러나 "특수 범주 중요한 데이터"로 볼 수 있는 데이터를 처리하는 경우 경우에 따라 데이터 처리를 기록해야 합니다. "가끔 처리"라는 정의는 모호하지만 한 번 또는 드물게 사용되는 데이터에 적용하기 위한 것입니다.
  
또한 수집한 개인 데이터가 보호되어 있는지도 확인해야 합니다. 즉, 암호화하고 적어도 암호를 사용하여 액세스 권한을 제어해야 합니다. 보호가 없는 데스크톱의 스프레드시트에 고객 데이터를 보관하면 GDPR의 기대치가 충족되지 않습니다. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>회사 웹 사이트가 GDPR을 준수하는지 어떻게 알 수 있나요?

첫 번째 질문은 사이트의 어디에서나 개인 데이터를 수집하나요? 예를 들어 이름과 전자 메일 주소를 묻는 연락처 양식이 있을 수 있습니다. 마케팅 전자 메일을 보내고 싶은 경우 데이터를 사용할 정확한 정보를 설명하는 '옵트인' 확인란을 추가해야 합니다. 받는 사람이 해당 확인란을 선택해야만 마케팅 목적으로 개인 데이터를 사용할 수 있습니다.
  
또한 데이터를 저장하는 데이터베이스가 보호되어 있는지 확인할 수 있습니다. 웹 호스팅 회사 또는 클라우드 저장소 공급업체에서 이에 대해 조언할 수 있습니다. 비즈니스용 Microsoft 365를 사용하는 경우 데이터 저장은 GDPR을 준수합니다. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>우리 회사는 유럽 외부에 있습니다. GDPR이 실제로 영향을 미치나요?

GDPR은 EU 시민을 보호하는 규정입니다. 회사에서 EU 시민을 상대하거나 향후에 원할 경우 영향을 받을 수 있습니다. 이는 EU 주에 거주하는 시민과 다른 곳에 거주하는 시민 모두에 적용됩니다.
  
다음 예제를 고려해 보십시오.
  
- EU 시민에게 자동차를 고용하는 미국 회사는 고객의 데이터를 수집하고 처리할 때 GDPR 요구 사항을 충족해야 합니다. 회사는 고객의 데이터를 사용할 때 동의를 얻어야하며 데이터가 안전하게 저장되도록 보장해야 합니다. 또한 고객이 모든 데이터 주체 권한을 적용할 수 있는지도 확인해야 합니다. 
    
- 오스트레일리아 회사에서 온라인으로 제품을 판매하고 해당 사용자는 온라인 계정을 설정합니다. GDPR 데이터 주체 권리 및 동의는 계정을 여는 EU 시민에게 적용됩니다. 회사는 고객이 모든 데이터 주체 권한을 적용할 수 있도록 해야 합니다. 
    
- 국제 자선 단체는 기조자에 대한 데이터를 수집하고 이를 사용하여 후원에 대한 업데이트 및 요청을 전송합니다. GDPR 상태: '... 직접 마케팅 목적으로 개인 데이터를 처리하는 경우 정당한 이익을 위해 이행된 것으로 생각될 수 있습니다." 그러나 조직의 책임은 데이터 주체의 관심사에 대해 다시 증명해야 합니다. 회사(또는 이 경우 자선 조직)는 항상 명시적인 옵트인 동의를 얻어야 합니다.
    
GDPR은 고객 데이터가 테두리를 넘어 이동하는 경우도 적용됩니다. 데이터 저장소에 클라우드 컴퓨팅을 사용하는 경우 서비스가 GDPR을 완전히 준수하는지 확인해야 합니다. 데이터 저장소가 데이터 보호 레코드가 낮은 위치에 있는 경우 복잡할 수 있습니다. 비즈니스용 Microsoft 365를 사용하는 경우 GDPR 요구 사항을 충족하기 위한 올바른 법적 설명서가 준비됩니다.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>물론 데이터를 수집하지만 다른 일부 회사에서는 데이터를 저장합니다. 후크를 해제할 수 있나요?

GDPR에서 데이터를 수집하는 경우 어느 정도 영향을 받을 수 있습니다. GDPR에는 데이터 프로세서와 데이터 컨트롤러의 개념이 있습니다. 
  
- **데이터 컨트롤러:** 데이터 수집 방법, 방법 및 이유가 결정되는 개인 또는 조직(공동 컨트롤러를 사용할 수 있습니다.) 다른 회사의 클라우드 서버를 사용하여 저장할 수 있습니다. 예를 들어 고객 데이터를 수집하는 웹 사이트는 컨트롤러입니다. 
    
- **데이터 프로세서:** 컨트롤러를 대신하여 데이터를 저장하고 요청 시 이러한 데이터를 처리하는 개인 또는 조직입니다. 예를 들어 비즈니스용 Microsoft 365 앱 데이터 저장소는 프로세서 역할을 하며 완전히 GDPR을 준수합니다. 
    
    조직 또는 시스템은 컨트롤러와 프로세서 모두의 역할을 할 수 있습니다. 비즈니스용 Microsoft 365는 GDPR을 준수하고 둘 다의 역할을 할 수 있습니다.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>이전 고객에게 마케팅 전자 메일을 보낼 수 있나요?

수년 동안 사용했던 고객도 마케팅에 데이터 사용에 동의하는지 확인해야 합니다. 이전에 동의를 캡처한 것일 수 있을 뿐만 아니라 이를 보여 주기 위해 레코드도 캡처한 것일 수 있습니다. 그렇다면 마케팅을 계속할 수 있습니다. 그렇지 않은 경우 고객에게 마케팅을 계속할 수 있는 권한을 고객에게 제공해야 합니다. 일반적으로 고객에게 사이트로 이동하여 향후 전자 메일을 받기 위한 동의 옵션을 선택하도록 요청하는 전자 메일을 보내는 과정입니다. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>신입 사원을 채용할 때 GDPR에 대해 걱정해야 하나요? 현재 직원은 어떻게 하나요?

GDPR은 고객 데이터에 영향을 주지 않습니다. 직원 데이터로도 확장됩니다. 신입사원은 종종 LinkedIn과 같은 소셜 미디어 플랫폼을 사용하여 신입사원을 모집합니다. 특정 사용 권한 없이 잠재적인 모집 데이터를 저장하지 않는지 확인
  
기존 직원과 신입사원 계약의 경우 계약 종료 시 서명이 반드시 동의할 필요는 없습니다( 특히 계약에 객관적이지 않은 조항이 사용되는 경우). 이 경우 조항과 연결된 명시적인 방식으로 동의를 캡처해야 합니다. 이는 직원 계약에 따라 달라지지만 경우에 따라 "적법한 이익"을 사용하여 직원 데이터 처리 공지를 추가하여 직원이 자신의 데이터로 할 작업을 인식할 수 있도록 할 수 있습니다.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>비즈니스용 Microsoft 365를 사용하여 개인 정보 보호 문제 충족

GDPR을 준수하는 것은 개인 데이터가 보호되어 있는지를 보장하는 것입니다. GDPR에는 디자인 및 기본값에 따라 개인 정보 보호라고 하는 개념이 있습니다. 즉, 개인 정보 보호를 충족하는 것이 두 번째 특성이 될 수 있도록 시스템 및 제품에 대한 데이터 보호가 "구체화"되어야 합니다. 
  
대기업의 경우와 마찬가지로 중소기업은 보안을 위해 편의를 제공해야 합니다. 비즈니스용 Microsoft 365는 직원 수가 300명 미만인 회사를 위해 디자인됩니다. 중소기업은 Microsoft 클라우드 기반 도구를 사용하여 비즈니스 생산성을 향상시킬 수 있습니다. 비즈니스용 Microsoft 365를 통해 중소기업은 전자 메일, 설명서 및 모임 및 이벤트를 관리할 수 있습니다. 또한 GDPR 규정 준수에 필수적인 기본 제공 보안 조치 및 장치 관리도 제공합니다.
  
비즈니스용 Microsoft 365는 다음과 같은 방식으로 GDPR 프로세스를 지원할 수 있습니다.
  
- **검색:** GDPR 규정 준수에 대한 중요한 단계는 현재 데이터를 아는 것입니다. 
    
- **관리:** 데이터에 대한 액세스를 제어하고 데이터 사용을 관리하는 것은 GDPR의 중요한 부분입니다. 비즈니스용 Microsoft 365는 장치에 적용하려는 정책에 따라 비즈니스 데이터를 보호합니다. 디바이스 관리는 직원이 원격으로 작업하는 시대에 중요합니다. 비즈니스용 Microsoft 365에는 모든 장치에서 데이터가 보호되는 장치 관리 기능이 포함되어 있습니다. 예를 들어, 비즈니스에 있는 모든 Windows 10 장치가 비즈니스용 Windows 10 장치를 통해 보호된다고 지정할 수 Windows Defender. 
    
- **보호:** 비즈니스용 Microsoft 365는 보안을 위해 디자인됩니다. 장치 관리 및 데이터 보호 컨트롤은 원격 장치를 포함하여 비즈니스 네트워크 전체에서 작동하여 데이터 보안을 유지하는 데 도움이 됩니다. 비즈니스용 Microsoft 365는 Office 응용 프로그램의 개인 정보 설정 및 문서 암호화와 같은 컨트롤을 제공합니다. 비즈니스용 Microsoft 365를 사용하면 GDPR 준수 모니터링을 수행하여 올바른 수준의 보호를 설정할 수 있습니다. 
    
- **보고서:** GDPR은 보고에 많은 강조를 합니다. 직원이 한 명인 기업에서도 많은 양의 데이터를 처리하면 절차를 문서화하고 보고해야 합니다. 비즈니스용 Microsoft 365는 소규모 조직에 대한 보고 요구 사항을 충족하지 않습니다. 
    
    감사 로그와 같은 도구를 사용하여 데이터 이동을 추적하고 보고할 수 있습니다. 보고서에는 수집 및 저장하는 데이터, 데이터로 무엇을 하는지, 데이터 전송을 분류하는 것이 포함됩니다. 
    
고객, 직원 및 클라이언트는 데이터 개인 정보 보호의 중요성을 더 잘 인식하고 있으며 이제 회사나 조직이 해당 개인 정보를 준수할 것으로 기대합니다. 비즈니스용 Microsoft 365는 비즈니스에 대한 대규모 업데이트 없이 GDPR 규정 준수를 달성하고 유지 관리하는 도구를 제공합니다.
  
## <a name="next-steps"></a>다음 단계

GDPR을 준비하기 위해 다음 단계에 대한 몇 가지 제안 사항이 있습니다.
  
- 책임 준비 검사 목록으로 GDPR [프로그램을 평가합니다.](/compliance/regulatory/gdpr-arc)
    
- GDPR 준수를 달성하고 유지하기 위한 솔루션으로 비즈니스용 [Microsoft 365를](https://www.microsoft.com/microsoft-365/business) 조사합니다.
