# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR 간소화: 소기업을 위한 가이드

 *비즈니스용 Microsoft 365를 사용하여 GDPR 규정 준수 완화 및 관리* 
  
GDPR(일반 데이터 보호 규정)은 조직이 개인 데이터를 처리하는 방법을 규정하는 EU(유럽 연합) 규정입니다. 비즈니스가 유럽 연합 시민에게 서비스를 제공하거나 유럽 연합 시민을 고용하는 경우 귀하는 [GDPR](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en)의 영향을 받습니다. 

소규모 기업 관리자는 "어떻게 시작해야 하나요?"라는 질문을 할 수 있습니다. 비즈니스가 개인 데이터를 핵심 비즈니스 활동으로 처리하지 않거나 GDPR을 처음 접해 본 경우 특히 그렇습니다.

이 문서를 검토하여 시작할 수 있습니다. 이 문서는 GDPR의 정의, GDPR이 제정된 이유 및 비즈니스용 Microsoft 365 조직이 GDPR을 준수하는 데 어떻게 도움이 되는지를 이해하는 데 도움이 됩니다.

또한 이 문서는 중소기업이 가질 수 있는 GDPR에 대한 일반적인 질문에 대한 답변과 중소기업이 GDPR에 대비하기 위해 취할 수 있는 단계를 중점적으로 설명합니다.

> [!IMPORTANT]
> 이 문서의 Microsoft 365 솔루션 및 권장 사항은 데이터를 관리하고 보호하는 데 도움이 될 수 있는 도구와 리소스이지만 GDPR 준수를 보장하지는 않습니다. 귀사의 규정 준수 상태 자체를 평가할 책임은 귀사에 달려 있습니다. 필요한 경우 사내 법률 및/또는 전문 자문가에게 문의하세요. 
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR에 대한 간략한 개요

GDPR은 1995년에 처음 시행된 이전 DPD(데이터 보호 지침)를 업데이트하고 확장하는 EU 규정입니다. GDPR은 개별 클라이언트, 고객, 직원 또는 비즈니스 파트너 등과 같은 개인 데이터의 개인 정보 보호와 관련이 있습니다. GDPR의 목표는 EU에 상주하는지 여부와 관계없이 EU 시민에 대한 개인 데이터 보호를 강화하는 것입니다. 이 규정은 기대치를 설정하고 이를 달성하는 방법에 대해 조언합니다. 조직에는 GDPR의 요구 사항을 충족하는 조치가 마련되어 있어야 합니다.
  
GDPR의 주요 내용은 데이터와 사용 방법에 관한 것입니다. 데이터에 수명 주기가 있다고 생각하세요. 이 주기는 데이터를 수집할 때 시작되고, 데이터를 저장하고 사용(처리)할 때 계속되며, 시스템에서 완전히 삭제하면 종료됩니다. 
  
GDPR은 다음 유형의 데이터와 관련이 있습니다. 
  
- **개인 데이터:** 데이터를 개인에 연결하고 식별할 수 있는 경우 해당 데이터는 GDPR과 관련하여 개인 데이터로 간주됩니다. 개인 데이터의 예로는 이름, 주소, 생년월일 및 IP 주소가 있습니다. GDPR은 데이터를 개인에 연결할 수 있는 경우 데이터가 얼마나 모호하거나 기술적인지에 관계없이 인코딩된 정보("가명 정보"라고도 함)도 개인 데이터로 간주합니다.
    
- **중요한 개인 데이터** 개인 데이터에 더 많은 세부 정보를 추가하는 데이터입니다. 예로는 인종, 무역 조합 구성원, 인종 출처 등이 있습니다. 중요한 개인 데이터에는 생체 인식 데이터 및 DNA도 포함됩니다. GDPR에 따르면 중요한 데이터에는 개인 데이터보다 더 엄격한 보호 규칙이 적용됩니다. 
    
## <a name="gdpr-terms"></a>GDPR 용어

다음에는 GDPR에서 자주 참조되는 몇 가지 용어가 표시됩니다. 이러한 용어를 이해하는 것이 중요합니다.
  
 **동의**
  
GDPR은 "개인 데이터의 처리는 인간에게 서비스를 제공하도록 설계되어야 ㄷ한다"라고 명시합니다. GDPR은 개인 데이터를 처리할 때 동의를를 통해 이러한 목표를 달성하려고 합니다. 해당 작업은 고객에게 회사로부터 이메일 메시지를 받을 것인지 묻는 간단한 작업일 수 있습니다. 또한 마케팅에 데이터를 사용하려는 경우 웹 사이트에 더 이상 옵트아웃 확인란이 없음을 의미합니다. "명확한 확인 행위"를 사용하여 명시적인 동의를 얻어야 합니다. 또한 동의를 취하거나 해지할 때에 관한 기록도 유지해야 합니다.
  
 **데이터 주체 권리**
  
GDPR은 데이터 주체 권한을 설정합니다. 즉, 고객, 직원, 비즈니스 파트너, 클라이언트, 계약자, 학생, 공급자 등은 그들의 개인 데이터에 대해 다음을 수행할 수 있는 권한이 있습니다.
  
- **데이터에 대한 알림:** 개인에게 데이터 사용에 대해 알려야 합니다. 
    
- **데이터에 대한 액세스 권한 보유:** 사용자가 보유한 데이터에 대한 액세스 권한을 개인에게 부여해야 합니다(예: 계정 액세스를 사용하거나 수동 방식으로). 
    
- **데이터 수정 요청:** 개인은 부정확한 데이터를 수정하도록 요청할 수 있습니다. 
    
- **삭제할 데이터 요청:** '삭제권'이라고도 하는 이 권리를 통해 개인은 회사에서 수집한 개인 데이터를 사용하거나 공유하는 모든 시스템에서 삭제하도록 요청할 수 있습니다. 
    
- **제한된 처리 요청:** 개인은 데이터를 표시하지 않거나 제한하도록 요청할 수 있습니다. 그러나 이 권리는 특정 상황에서만 적용할 수 있습니다. 
    
- **데이터 이동성 보유:** 개인이 데이터를 다른 회사로 전송하도록 요청할 수 있습니다. 
    
- **개체:** 개인은 직접 마케팅을 포함하여 다양한 용도로 사용되는 데이터에 개체를 지정할 수 있습니다. 
    
- **프로파일링을 포함하여 자동화된 의사 결정의 적용을 받지 않도록 요청:** GDPR에는 데이터를 사용하여 사용자를 프로파일링하고 해당 프로파일링에 따라 의사 결정을 자동화하는 행위에 대한 엄격한 규칙이 있습니다. 


## <a name="steps-to-prepare-for-gdpr"></a>GDPR 준비 단계

이 섹션에서는 중소기업이 GDPR을 준비하기 위해 수행할 수 있는 단계에 대해 설명합니다. 이러한 단계에 대한 많은 정보는 유럽 연합의 간행물 사무소를 통해 제공되는 간행물인 [기업이 일반 데이터 보호 규정을 준비하기위한 7단계](https://ec.europa.eu/info/sites/default/files/ds-02-18-544-en-n.pdf)를 통해 제공되었습니다.

중소기업이 GDPR을 시작하는 좋은 방법은 개인 데이터를 수집할 때 다음 주요 원칙을 적용하는 것입니다.

- 사용하는 용도에 대해 명확하게 정의된 목적으로 개인 데이터를 수집하고 다른 용도로 사용하지 마세요. 예를 들어 고객에게 새로운 제안이나 프로모션을 받을 수 있도록 이메일 주소를 제공하라고 요청하는 경우 특정 목적으로만 이메일 주소를 사용할 수 있습니다.
- 필요한 것보다 많은 데이터를 수집하지 마세요. 예를 들어, 비즈니스에서 상품을 배달하기 위해 우편 주소가 필요한 경우 고객의 주소와 이름이 필요하지만 사용자의 혼인 상태를 알 필요는 없습니다.


### <a name="step-1-know-the-personal-data-that-you-collect-and-use-within-your-business-and-the-reasons-you-need-it"></a>1단계: 비즈니스 내에서 수집하고 사용하는 개인 데이터와 필요한 이유 파악

중소기업의 첫 번째 단계 중 하나는 비즈니스 내에서 수집하고 사용하는 개인 데이터의 인벤토리를 만들고 해당 데이터가 필요한 이유를 만드는 것입니다. 여기에는 직원과 고객 모두에 대한 데이터가 포함됩니다.

예를 들어 고용 계약 및 법적 이유(예: 국세청 세금 보고)에 따라 직원의 개인 데이터가 필요할 수 있습니다.

또 다른 예로, 개별 고객이 이에 동의한 경우 특별 제안에 대한 알림을 보내도록 개별 고객 목록을 관리할 수 있습니다.

#### <a name="microsoft-365-features-that-can-help"></a>도움이 될 수 있는 Microsoft 365 기능
[Microsoft 365의 Microsoft Information Protection](/microsoft-365/compliance/information-protection)은 회사에서 중요한 정보를 검색, 분류 및 보호하는 데 도움이 될 수 있습니다. 학습 가능한 분류자를 사용하여 개인 데이터가 포함된 문서 형식을 식별하고 레이블을 지정할 수 있습니다. 

### <a name="step-2-inform-your-customers-employees-and-other-individuals-when-you-need-to-collect-their-personal-data"></a>2단계: 개인 데이터를 수집해야 하는 경우 고객, 직원 및 기타 개인에게 알립니다.

개인은 개인 데이터와 용도를 처리한다는 것을 알고 있어야 합니다. 예를 들어 고객이 비즈니스의 온라인 사이트에 액세스하기 위해 고객 프로필을 만들어야 하는 경우 해당 정보로 수행할 작업을 구체적으로 명시해야 합니다.

그러나 사용자가 데이터를 사용하는 방법을 이미 알고 있는 경우에는 개인에게 알릴 필요가 없습니다. 예를 들어, 그들이 주문한 배달을 위해 집 주소를 제공할 때가 해당됩니다.

또한 사용자가 보유하고 있는 개인 데이터에 대한 요청에 대해 개인에게 알리고 해당 데이터에 대한 액세스 권한을 부여할 수 있어야 합니다. 데이터를 체계적으로 관리하면 필요한 경우 해당 데이터를 더 쉽게 제공할 수 있습니다. 

### <a name="step-3-keep-personal-data-for-only-as-long-as-necessary"></a>3단계: 필요한 기간 동안만 개인 데이터 유지

직원 데이터의 경우 고용 관계가 유지되고 관련 법적 의무가 있는 한 해당 데이터를 보관합니다.
고객 데이터의 경우 고객 관계가 유지되고 관련 법적 의무(예: 세무 목적)가 있는 한 해당 데이터를 보관합니다.
수집하는 목적에 더 이상 필요하지 않은 데이터를 삭제하세요.

#### <a name="microsoft-365-features-that-can-help"></a>도움이 될 수 있는 Microsoft 365 기능
[보존 정책 및 레이블](/microsoft-365/compliance/retention)을 사용하여 특정 시간 동안 개인 데이터를 유지하고 더 이상 필요하지 않은 경우 삭제할 수 있습니다.


### <a name="step-4-secure-the-personal-data-you-are-processing"></a>4단계: 처리 중인 개인 데이터 보호

IT 시스템에 개인 데이터를 저장하는 경우 데이터가 포함된 파일에 대한 액세스를 제한하세요(예: 강력한 암호를 통해). 시스템의 보안 설정을 정기적으로 업데이트합니다.

> [!NOTE]
> GDPR은 특정 IT 시스템의 사용을 규정하지 않지만 시스템에 적절한 수준의 보안을 갖도록 합니다. 자세한 내용은 [GDRP 32조: 처리 보안](https://gdpr.eu/article-32-security-of-processing/)을 참조하세요.

개인 데이터를 사용하여 물리적 문서를 저장하는 경우 권한이 없는 사용자가 액세스할 수 없는지 확인합니다.

Microsoft 365와 같은 클라우드에 개인 데이터를 저장하도록 선택하는 경우 파일 및 폴더에 대한 사용 권한 관리 기능, 파일을 저장할 중앙 집중식 보안 위치(OneDrive 또는 SharePoint 문서 라이브러리), 파일을 보내거나 검색할 때 데이터 암호화 등의 보안 기능이 있습니다. 

#### <a name="microsoft-365-features-that-can-help"></a>도움이 될 수 있는 Microsoft 365 기능
[Microsoft DLP(데이터 손실 방지)](/microsoft-365/business-video/set-up-dlp)를 사용하여 비즈니스의 중요한 정보를 보호할 수 있습니다. [GDPR 템플릿](/microsoft-365/compliance/what-the-dlp-policy-templates-include#general-data-protection-regulation-gdpr)을 사용하는 [DLP 정책을 설정](/microsoft-365/compliance/create-a-dlp-policy-from-a-template)할 수도 있습니다.

### <a name="step-5-keep-documentation-on-your-data-processing-activities"></a>5단계: 데이터 처리 활동에 대한 설명서 유지

보유하고 있는 개인 데이터와 그 이유를 설명하는 간단한 문서를 작성합니다. 필요한 경우 국가 데이터 보호 기관에 설명서를 제공해야 할 수 있습니다.

이러한 문서에는 아래에 나열된 정보가 포함되어야 합니다.

| 정보 | 예제 |
|---|---|
|데이터 처리의 목적|자택 배송, 공급자에게 지불, 직원을 위한 급여 및 사회 보장 보장과 같은 특별한 제공에 대해 고객에게 알림|
|개인 데이터 형식|고객의 연락처 세부 정보; 공급자의 연락처 세부 정보; 직원 데이터|
|관련된 데이터 주체의 범주|직원; 고객, 공급 업체|
|받는 사람의 범주|근로 기관; 세금 기관|
|저장 기간|고용 계약이 종료될 때까지 직원의 개인 데이터(및 관련 법적 의무) 고객/계약 관계가 끝날 때까지 고객의 개인 데이터|
|개인 데이터를 보호하기 위한 기술 및 조직 보안 조치|IT 시스템 솔루션을 정기적으로 업데이트, 안전한 위치, 액세스 제어, 데이터 암호화, 데이터 백업|
|개인 데이터가 EU 외부의 받는 사람에게 전송되는지 여부|EU 외부의 프로세서 사용(예: 클라우드의 저장소) 프로세서의 데이터 위치; 계약 약정|

</br>

Microsoft의 개인 정보 보호 및 보안 약정, 데이터 처리 약관 및 고객이 볼륨 라이선싱 계약에 따라 구독하는 Microsoft 호스팅 서비스에 대한 GDPR 조건을 제공하는 [Microsoft Online Services Data Protection 부록](https://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=2&Keyword=DPA)에서 GDPR과 관련된 Microsoft의 계약 약정을 확인할 수 있습니다. 


### <a name="step-6-make-sure-your-subcontractors-respect-the-rules"></a>6단계: 하도급업자가 규칙을 준수하는지 확인

개인 데이터의 처리를 다른 회사에 하위 계약하는 경우 GDPR의 요구 사항(예: 보안 조치)에 따른 처리를 보장하는 서비스 공급자만 사용하세요. 



### <a name="step-7-assign-someone-to-oversee-personal-data-protection"></a>7단계: 개인 데이터 보호를 감독할 사용자 할당
 
개인 데이터를 더 잘 보호하기 위해 조직에서 <b>DPO(데이터 보호 책임자)</b>를 임명해야 할 수도 있습니다. 그러나 개인 데이터 처리가 비즈니스의 핵심 부분이 아니거나 중소기업인 경우 데이터 보호 책임자를 지정할 필요가 없습니다. 예를 들어, 귀사에서 자택 배송을 위해서만 고객의 데이를 수집하는 경우에는 DPO를 지정할 필요가 없습니다. DPO를 사용해야 하는 경우에도 이러한 의무는 다른 작업과 더불어 기존 직원에게 할당될 수 있습니다. 또는 필요에 따라 이 업무를 위해 외부 컨설턴트를 고용하도록 선택할 수 있습니다.

일반적으로 [데이터 보호 영향 평가](https://gdpr.eu/article-35-impact-assessment/)를 수행할 필요가 없습니다. 이는 개인 데이터에 더 많은 위험을 초래하는 기업에 대해 유보되어 있습니다(예: 비디오 감시와 같이 공개적으로 액세스할 수있는 영역에 대한 대규모 모니터링을 수행하는 경우).

직원 임금과 고객 목록을 관리하는 중소기업인 경우 일반적으로 데이터 보호 영향 평가를 수행할 필요가 없습니다.  
 

    
## <a name="common-small-business-questions-about-the-gdpr"></a>GDPR에 대한 일반적인 중소기업 질문

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>저는 개인 사업자입니다. GDPR에 대해 정말로 신경써야 하나요?

GDPR에서 중요한 것은 보유한 직원 수가 아니라 처리하는 데이터입니다. GDPR은 모든 규모의 회사, 심지어 개인 사업자에게 영향을 미칩니다. 그러나 직원 수가 250명 미만인 기업은 기록 보관 감소와 같은 일부 면제 혜택을 받지만, 데이터 처리가 개인의 권리에 영향을 미치지 않고 가끔씩 처리되고 있다고 확신하는 경우에만 적용됩니다.
  
예를 들어, 개인 데이터가 아닌 데이터에 대한 처리는 면제되거나 축소된 조치가 필요합니다. 그러나 "특별한 범주에 민감한 데이터"로 간주되는 데이터를 처리하는 경우에는 발생 빈도가 적더라도 이러한 데이터 처리를 기록해야 합니다. "비 정기적 처리"의 정의는 모호하지만 한 번 또는 드물게 사용되는 데이터에 적용하기 위한 것입니다.
  
또한 수집하는 개인 데이터가 보호되는지 확인해야 합니다. 즉, 암호를 암호화하고 최소한 암호를 사용하여 액세스를 제어해야 합니다. 보호 조치 없이 고객 데이터를 데스크톱의 스프레드시트에 보관하는 것은 GDPR 기대 수준을 충족하지 못합니다. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>회사 웹 사이트가 GDPR을 준수하는지 어떻게 알 수 있나요?

스스로 물어볼 첫 번째 질문은 ‘사이트의 어느 위치에서나 개인 데이터를 수집하는가?’입니다. 예를 들어 이름과 이메일 주소를 묻는 문의 양식이 있을 수 있습니다. 마케팅 이메일을 보내려면 데이터를 정확히 어떤 용도로 사용할 것인지를 설명하는 '옵트인' 확인란을 추가해야 합니다. 수신자가 해당 확인란을 선택한 경우에만 마케팅 목적으로 개인 데이터를 사용할 수 있습니다.
  
또한 데이터를 저장하는 데이터베이스가 보호되는지 확인하세요. 웹 호스팅 회사 또는 클라우드 저장소 공급자가 이러한 사항에 대해 정보를 제공할 수 있습니다. 비즈니스용 Microsoft 365를 사용하는 경우 데이터 저장소는 GDPR을 준수합니다. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>우리 회사는 유럽 밖에 있습니다. GDPR이 실제로 우리 회사에게 영향을 미치나요?

GDPR은 EU 시민을 보호하는 규정입니다. 회사가 지금 EU 시민을 상대하거나 미래에 그러기를 희망하는 경우 GDPR의 영향을 받을 것입니다. 이는 EU 국가에 거주하는 시민과 다른 국가에 거주하는 시민 모두에게 적용됩니다.
  
다음 예제를 고려하세요.
  
- EU 시민에게 자동차를 대여하는 미국 회사는 고객의 데이터를 수집하고 처리할 때 GDPR 요구 사항을 충족해야 합니다. 회사는 고객의 데이터를 수집할 때 동의를 얻어 데이터가 안전하게 저장되도록 해야 합니다. 또한 고객이 모든 데이터 주체 권한을 적용할 수 있는지 확인해야 합니다. 
    
- 한 호주 회사가 온라인으로 제품을 판매하고 사용자는 온라인 계정을 설정합니다. GDPR 데이터 주체 권한 및 동의는 계정을 개설 한 EU 시민에게 적용됩니다. 회사는 고객이 모든 데이터 주체 권한을 적용할 수 있는지 확인해야 합니다. 
    
- 국제 자선 단체는 기부자에 대한 데이터를 수집하고 이를 사용하여 업데이트 및 기부 요청을 보냅니다. GDPR에는 다음과 같이 명시되어 있습니다. “...직접 마케팅 목적으로 개인 데이터를 처리하는 것은 정당한 이익을 위해 수행되는 것으로 간주될 수 있습니다." 그러나 회사의 이익이 데이터 주체의 이익보다 우선한다는 것을 증명할 책임은 조직에 있습니다. 회사(또는 이 경우 자선단체)는 항상 정보에 대한 명시적인 옵트인 동의를 받아야 합니다.
    
GDPR은 고객 데이터가 국경을 넘어 이동하는 경우에도 적용됩니다. 데이터 저장에 클라우드 컴퓨팅을 사용하는 경우 서비스가 GDPR을 완전히 준수하는지 확인해야 합니다. 데이터 보호 기록이 좋지 않은 위치에 데이터 저장소가 위치할 경우, 이 작업은 복잡해질 수 있습니다. 비즈니스용 Microsoft 365를 사용하는 경우 GDPR 요구 사항을 충족할 수 있는 올바른 법적 문서가 있습니다.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>물론입니다. 데이터를 수집하지만 다른 회사에서 저장합니다. 그러면 GDPR의 적용을 받지 않게 되나요?

GDPR에 따라 데이터를 수집하면 어느 정도 영향을 받습니다. GDPR에는 데이터 프로세서와 데이터 컨트롤러의 개념이 있습니다. 
  
- **데이터 컨트롤러:** 데이터 수집 방법, 내용 및 이유를 결정하는 개인 또는 조직(공동 컨트롤러를 둘 수 있음). 이들은 다른 회사의 클라우드 서버를 사용하여 저장할 수 있습니다. 예를 들어 고객 데이터를 수집하는 웹 사이트는 컨트롤러입니다. 
    
- **데이터 프로세서:** 컨트롤러를 대신하여 데이터를 저장하고 요청시 이러한 데이터를 처리하는 개인 또는 조직입니다. 예를 들어 Microsoft 365 Apps for business 데이터 저장소는 프로세서 역할을하며 GDPR을 완전히 준수합니다. 
    
    조직이나 시스템은 컨트롤러와 프로세서의 역할을 모두 수행할 수 있습니다. 비즈니스용 Microsoft 365는 둘 다로 작동할 수 있으며 GDPR을 준수합니다.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>이전 고객에게 마케팅 이메일을 보낼 수 있나요?

고객, 심지어 수년 동안 보유해 온 고객도 고객의 데이터를 마케팅에 사용하기로 동의했는지 확인해야 합니다. 과거에 동의와 이를 보여주는 기록을 캡처했을 수 있습니다. 그렇다면 마케팅을 계속할 준비가 된 것입니다. 그렇지 않은 경우 고객을 대상으로 마케팅 활동을 계속하려면 고객의 허가를 받아야 합니다. 여기에는 일반적으로 고객에게 사이트로 이동하여 향후 이메일 수신에 동의하는 옵션을 선택하도록 요청하는 이메일을 보내는 것이 포함됩니다. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>신규 직원을 채용할 때 GDPR에 대해 걱정해야 하나요? 현재 직원의 경우에는 어떤가요?

GDPR은 고객 데이터에만 영향을 주지 않습니다. 직원 데이터에도 확장 적용됩니다. 신입 사원은 종종 LinkedIn과 같은 소셜 미디어 플랫폼을 사용하여 배치됩니다. 명시적 허가 없이 잠재적인 채용 데이터를 저장하지 마세요.
  
기존 근로자와 신규 고용계약의 경우 계약 종료 시 서명이 반드시 동의하는 것은 아니며, 특히 계약에서 비확정 조항이 사용되는 경우에는 더욱 그렇습니다. 이 경우 조항과 관련된 명시적인 방식으로 동의를 확보해야 합니다. 이것이 의미하는 바는 직원 계약에 따라 다르지만, 경우에 따라 "합법적인 이익"을 사용하고 직원 데이터 처리 통지를 추가하여 직원이 해당 데이터에 대해 무엇을 할 것인지 확실히 알 수 있습니다.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>비즈니스용 Microsoft 365를 사용하여 개인 정보 문제 해결

GDPR을 준수하는 것은 개인 데이터를 보호하는 것입니다. GDPR에는 디자인 및 기본에 의한 개인 정보 보호라는 개념이 있습니다. 즉, 개인 정보 보호 문제를 해결하는 것이 제2의 특성이 될 수 있도록 데이터 보호 기능을 시스템과 제품에 "반영"해야 합니다. 
  
규모가 큰 기업과 마찬가지로 중소기업도 보안을 저해시키지 않고 편리함을 추구할 필요가 있습니다. 비즈니스용 Microsoft 365는 직원 수가 300명 미만인 회사를 위해 설계되었습니다. 소규모 회사는 Microsoft 클라우드 기반 도구를 사용하여 비즈니스 생산성을 향상시킬 수 있습니다. 비즈니스용 Microsoft 365를 사용하면 소규모 비즈니스에서 이메일, 문서는 물론 회의와 이벤트까지 관리할 수 ​​있습니다. 또한 GDPR 규정 준수에 필수적인 보안 조치 및 장치 관리가 내장되어 있습니다.
  
비즈니스용 Microsoft 365는 다음과 같은 방법으로 GDPR 프로세스를 지원할 수 있습니다.
  
- **검색:** GDPR 준수를 위한 중요한 단계는 보유한 데이터를 아는 것입니다. 
    
- **관리:** 데이터에 대한 액세스 제어 및 데이터 사용 관리는 GDPR의 필수 부분입니다. 비즈니스용 Microsoft 365는 장치에 적용하려는 정책을 기반으로 비즈니스 데이터를 보호합니다. 직원이 원격으로 작업하는 시대에 장치 관리는 매우 중요합니다. 비즈니스용 Microsoft 365에는 모든 장치에서 데이터를 보호하는 장치 관리 기능이 포함되어 있습니다. 예를 들어 비즈니스의 모든 Windows 10 장치가 Windows Defender를 통해 보호되도록 지정할 수 있습니다. 
    
- **보호:** 비즈니스용 Microsoft 365는 보안을 위해 설계되었습니다. 장치 관리 및 데이터 보호 제어는 원격 장치를 포함한 비즈니스 네트워크에서 작동하여 데이터를 안전하게 유지하는 데 도움이됩니다. 비즈니스용 Microsoft 365는 Office 응용 프로그램의 개인 정보 설정 및 문서 암호화와 같은 제어 기능을 제공합니다. 비즈니스용 Microsoft 365를 사용하면 GDPR 규정 준수 모니터링을 수행하여 적절한 수준의 보호 집합이 있는지 확인할 수 있습니다. 
    
- **보고서:** GDPR은 보고 활동에 중점을 둡니다. 직원이 한 명인 기업이라도 많은 양의 데이터를 처리하는 경우 절차를 문서화하고보고해야 합니다. 비즈니스용 Microsoft 365는 소규모 조직에 대한 보고 요구 사항의 번거로움을 해소합니다. 
    
    감사 로그와 같은 도구를 사용하면 데이터 이동을 추적하고 보고할 수 있습니다. 보고서에는 수집 및 저장하는 데이터 분류, 데이터로 수행하는 작업 및 데이터 전송이 포함됩니다. 
    
고객, 직원 및 클라이언트는 데이터 개인 정보 보호의 중요성을 더욱 심각하게 인식하고 있으며 이제 회사 또는 조직이 해당 개인 정보를 존중하기를 기대합니다. 비즈니스용 Microsoft 365는 기업에게 큰 변화없이 GDPR 규정 준수를 달성하고 유지하는 도구를 제공합니다.
