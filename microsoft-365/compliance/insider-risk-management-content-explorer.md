# <a name="insider-risk-management-content-explorer"></a>내부자 위험 관리 콘텐츠 탐색기

내부자 위험 관리 콘텐츠 탐색기를  사용하면 내부자 위험 관리 조사자 역할이 할당된 사용자가 경고의 활동과 관련된 콘텐츠의 컨텍스트 및 세부 정보를 검사할 수 있습니다. 사례로 확인된 모든 경고의 경우 데이터 및 메시지 파일의 복사본은 항목의 시간 동안 스냅숏으로 보관되는 동시에 원본 파일 및 메시지를 저장소 원본에 유지 관리합니다. 데이터 및 메시지의 복사는 경고와 연결된 사용자 및 콘텐츠 소유자에게 투명합니다. 콘텐츠에 정보 권한 관리 권한이 포함된 경우 복사된 콘텐츠에 대해 이러한 사용  권한이 유지 관리되는 것이고 내부자 위험 관리 조사자 역할이 할당된 사용자는 파일을 열고 보아야 하는 경우 이러한 사용 권한 및 권한이 필요합니다. 각 파일 및 메시지에는 관리 목적으로 내부자 위험 관리 사례에서 고유한 파일 ID가 자동으로 할당됩니다. 장치 표시기 활동과 관련된 문서는 콘텐츠 탐색기에서 포함되지 않습니다.

## <a name="column-options"></a>열 옵션

위험 분석가와 조사자는 캡처된 데이터 및 메시지를 보다 쉽게 검토하고 사례에 대한 컨텍스트를 검토할 수 있도록 콘텐츠 탐색기에 여러 가지 필터링 및 정렬 도구가 포함되어 있습니다. 기본 정렬의 경우 **날짜** 및 파일 클래스 열은 콘텐츠 큐 창의 열 제목을 사용하여 정렬을 지원합니다.  다른 큐 열을 보기에 추가하여 파일 및 메시지에 다른 피벗을 제공할 수 있습니다.

콘텐츠 큐의 열 제목을 추가하거나 제거하려면 열 편집 컨트롤을 **사용하여** 다음 열 옵션에서 선택합니다. 이러한 열은 콘텐츠 탐색기에서 지원되는 일반, 전자 메일 및 문서 속성 조건에 매핑되어 있으며 이 문서의 뒷부분에 나와 있습니다.

## <a name="advanced-search-conditions"></a>고급 검색 조건

검색 조건을 추가하여 검색 범위를 좁히고 보다 구체화된 결과 집합을 반환할 수 있습니다. 각 조건은 검색을 시작할 때 생성 및 실행되는 검색 쿼리에 절을 추가합니다. 조건은 AND 연산자와 기능과 비슷한 논리 연산자(c:c로 표현)에 의해 키워드 쿼리(키워드 상자에 지정)에 논리적으로 연결됩니다. 즉, 항목이 키워드 쿼리와 검색 결과에 포함될 하나 이상의 조건을 모두 충족해야 합니다. 이 기능은 조건이 결과 범위를 좁히는 데 도움이 되는 방식입니다.

고급 필터 및 검색 도구의 경우 콘텐츠 큐의 왼쪽에 있는 필터 창을 확장합니다.  조건 목록을 **열기** 위해 조건 추가 단추를 선택합니다.

### <a name="operators-used-with-conditions"></a>조건과 함께 사용되는 연산자

|**연산자**|**연산자와 동일한 쿼리**|**설명**|
|:-----------|:-------------------|:--------------|
| **After** |`property>date`| 날짜 조건과 함께 사용됩니다. 지정된 날짜 이후에 전송, 수신 또는 수정된 항목을 반환합니다.|
| **이전** |`property<date`| 날짜 조건과 함께 사용됩니다. 지정된 날짜 이전에 전송, 수신 또는 수정된 항목을 반환합니다.|
| **Between** |`date..date`| 날짜 및 크기 조건과 함께 사용됩니다. 날짜 조건과 함께 사용될 경우 지정된 날짜 범위 내에서 전송, 수신 또는 수정된 항목을 반환합니다. 크기 조건과 함께 사용될 경우 해당 크기가 지정된 범위 내에 속하는 항목을 반환합니다.|
| **모두 포함** |`(property:value) OR (property:value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정된 문자열 값을 모두 포함하는 항목을 반환합니다. |
| **Contains any of** |`(property:value) OR (property:value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정된 문자열 값의 일부를 포함하는 항목을 반환합니다.|
| **다음을 포함하지 않습니다.** |`-property:value`  <br/> `NOT property:value`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 지정된 문자열 값의 어떤 부분도 포함하지 않는 항목을 반환합니다.|
| **같지 않습니다.** |`-property=value`  <br/> `NOT property=value`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 특정 문자열을 포함하지 않는 항목을 반환합니다.|
| **같음** |`size=value`| 지정한 크기와 같은 항목을 반환합니다. <sup>1</sup>|
| **Equals any of** |`(property=value) OR (property=value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정된 문자열 값과 정확히 일치하는 항목을 반환합니다.|
| **같음 없음** |`(property=value) OR (property=value)`| 문자열 값을 지정하는 속성에 대한 조건과 함께 사용됩니다. 하나 이상의 지정한 문자열 값과 일치하지 않는 항목을 반환합니다. |
| **보다 큼** |`size>value`| 지정한 속성이 지정된 값보다 큰 항목을 반환합니다. <sup>1</sup>|
| **Greater or equal** |`size>=value`| 지정한 속성이 지정된 값보다 크거나 같은 항목을 반환합니다. <sup>1</sup>|
| **보다 작음** |`size<value`| 특정 값보다 크거나 같은 항목을 반환합니다. <sup>1</sup>|
| **Less or equal** |`size<=value`| 특정 값보다 크거나 같은 항목을 반환합니다. <sup>1</sup>|
| **Not equal** |`size<>value`| 지정한 크기와 같지 않은 항목을 반환합니다. <sup>1</sup>|

> [!NOTE]
> <sup>1</sup> 이 연산자는 Size 속성을 사용하는 조건에만 사용할 수 있습니다.

| **모임 종료 날짜** | 모임의 모임 종료 날짜입니다. |
| **모임 시작 날짜** | 모임의 모임 시작 날짜입니다. |
| **메시지 종류** | 검색할 전자 메일 메시지의 유형입니다. 가능한 값: 연락처, docs, 전자 메일, 외부 데이터, 팩스, im, 업무 일지, 모임, Microsoft 팀(Microsoft Teams에서 채팅, 모임 및 통화에서 항목 반환), 메모, 게시물, rssfeeds, 작업, 음성 메일 |
| **참가자 도메인** | 메시지 참가자의 모든 도메인 목록입니다. |
| **참가자** | 전자 메일 메시지의 모든 사용자 필드 이러한 필드는 From, To, Cc 및 Bcc입니다. |
| **Received** | 받는 사람이 전자 메일 메시지를 받은 날짜입니다. |
| **받는 사람 도메인** | 메시지 받는 사람의 모든 도메인 목록입니다. |
| **보낸 사람** | 메시지 유형에 대한 보낸 사람(보낸 사람) 필드입니다.  Format은 **DisplayName입니다. \<SmtpAddress>** |
| **보낸 사람 도메인** | 보낸 사람 도메인입니다. |
| **제목** | 전자 메일 메시지 제목 줄의 텍스트입니다.  <br/> **참고:** 쿼리에서 Subject 속성을 사용하면 제목 줄에 검색할 텍스트가 포함된 모든 메시지가 반환됩니다. 즉, 쿼리는 정확히 일치하는 메시지만 반환하지는 않습니다. 예를 들어 검색하는 경우 결과에 `subject:"Quarterly Financials"` 제목이 "Quarterly Financials 2018"인 메시지가 포함됩니다. |
| **To** | 전자 메일 메시지의 받는 사람 필드입니다. |
| **전자 메일 집합의 고유** | 전자 메일 집합에 첨부 파일이 중복되어 있는 경우 False입니다. |

## <a name="document-property-conditions"></a>문서 속성 조건

다음 표에는 콘텐츠 탐색기를 사용할 수 있는 문서 속성 조건이 나열되어 있습니다. 이러한 속성 조건 중 다수는 [Advanced eDiscovery](document-metadata-fields-in-Advanced-eDiscovery.md)사례에 포함된 검토 집합과 공유됩니다.

| **조건 옵션** | **설명** |
|:---------------------|:----------------|
| **변호사-클라이언트 권한 점수** | 변호사 클라이언트 권한 모델 콘텐츠 점수입니다. |
| **만든 이** | 문서를 복사하는 경우 유지되는 Office 문서의 만든 이 필드입니다. 예를 들어 사용자가 문서를 만든 후 다른 사람에게 전자 메일을 보내 SharePoint에 업로드하는 경우 문서는 여전히 원본 작성자를 보존합니다. |
| **규정 준수 레이블** | Office 365에 적용된 준수 레이블입니다. |
| **복합 경로** | 항목의 원본을 설명하는 사람이 읽을 수 있는 경로입니다. |
| **대화 ID** | 메시지의 대화 ID입니다. |
| **만든 시간** | 파일 또는 전자 메일 메시지를 만든 시간입니다. |
| **Custodian** | 항목이 연결된 보전자 이름입니다. |
| **주된 테마** | 분석에 대해 계산된 주 테마입니다. |
| **가족 ID** | 가족 ID는 모든 항목을 그룹화합니다. 전자 메일의 경우 이 필드에는 메시지와 모든 첨부 파일이 포함됩니다. 문서의 경우 이 필드에는 문서와 포함된 항목이 포함됩니다. |
| **파일 클래스** | SharePoint 및 OneDrive의 콘텐츠: **문서;** For content from Exchange: **Email or **Attachment**. |
| **파일 형식** | 파일의 확장명입니다. 예를 들어 docx, one, pptx 또는 xlsx와 같습니다. |
| **변호사 참가자가 있습니다.** | True이면 참가자 중 한 명 이상이 변호사 목록에 있습니다. 그렇지 않으면 값이 False입니다. |
| **Immutable ID** | Office 365에 저장된 Immutable ID입니다. |
| **포괄 유형** | 분석에 대해 계산된 포함 유형: **0** - 포괄이 아 않습니다. **1** - 포함; **2** - 포괄 minus; **3** - 포괄 복사본. |
| **Item 클래스** | Exchange 서버에서 제공하는 항목 클래스 예를 들어 **IPM과 같습니다. 참고** |
| **마지막으로 수정한 날짜** | 문서를 마지막으로 변경한 날짜입니다. |
| **로드 ID** | 항목이 검토 집합에 로드된 로드 ID입니다. |
| **위치 이름** | 항목의 원본을 식별하는 문자열입니다.  Exchange의 경우 이 필드는 사서함의 SMTP 주소가 됩니다. SharePoint 및 OneDrive의 경우 사이트 모음의 URL입니다. |
| **대표로 표시** | 정확히 중복된 각 집합의 문서 한 개가 담당자로 표시됩니다. |
| **기본 파일 확장명** | 항목의 기본 확장명입니다. |
| **기본 파일 이름** | 항목의 기본 파일 이름입니다. |
| **NdEtSortExclAttach** | 검토 시 효율적인 정렬을 위해 전자 메일 집합 및 ND 집합을 함께 사용할 수 있습니다. D는 ND 집합에 대한 prefix로 추가하고 E는 전자 메일 집합에 추가됩니다. |
| **피벗 ID** | 피벗의 ID입니다. |
| **권한이 부여될 수 있습니다.** | True이면 변호인 권한 검색 모델이 잠재적으로 권한이 부여된 문서를 고려합니다. |
| **처리 상태** | 항목이 검토 집합에 추가된 후의 처리 상태입니다. |
| **읽기 백분위수** | 관련성에 따라 문서에 대한 읽기 백분위수입니다. |
| **타당성 점수** | 관련성 기반 문서의 관련성 점수입니다. |
| **타기지사 태그** | 관련성에 따라 문서의 관련성 점수입니다. |
| **대표 ID** | 각 중복 항목 집합의 숫자 식별자입니다. |
| **태그** | 검토 집합에 적용된 태그입니다. |
| **테마 목록** | 분석에 대해 계산된 테마 목록입니다. |
| **Title** | 문서의 제목입니다. Title 속성은 Office 문서에 지정된 메타데이터입니다. 문서의 파일 이름과 다릅니다. |
| **수정된 경우** | 항목이 수정된 경우 True로, 그렇지 않으면 False입니다. |
| **단어 개수** | 파일의 단어 수입니다. |
