# <a name="insider-risk-management-content-explorer"></a>내부자 위험 관리 콘텐츠 탐색기

내부자 위험 관리 **콘텐츠** 탐색기를  사용하면 내부자 위험 관리 조사자 역할이 할당된 사용자가 경고의 활동과 관련된 콘텐츠의 컨텍스트 및 세부 정보를 검사할 수 있습니다. 콘텐츠 탐색기에서 사례 데이터는 새 활동을 포함하기 위해 매일 새로 고쳐집니다. 사례로 확인된 모든 경고의 경우 데이터 및 메시지 파일의 복사본은 항목의 시간 동안 스냅숏으로 보관되는 동시에 원본 파일 및 메시지를 저장소 원본에 유지 관리합니다. 필요한 경우 사례 데이터 파일을 PDF(이동식 문서 파일) 또는 원본 파일 형식으로 내보낼 수 있습니다.

새 경우 일반적으로 콘텐츠 탐색기에서 콘텐츠를 채우는 데 1시간 정도 소요됩니다. 콘텐츠가 많은 경우 스냅숏을 만드는 데 시간이 오래 걸릴 수 있습니다. 콘텐츠가 콘텐츠 탐색기에서 계속 로드되는 경우 완료율을 표시하는 진행률 표시기가 표시됩니다.

경우에 따라 사례와 연결된 데이터를 콘텐츠 탐색기에서 검토하기 위한 스냅숏으로 사용할 수 없는 경우도 있습니다. 이러한 상황은 사례 데이터를 삭제하거나 이동하거나 사례 데이터를 처리하는 경우 일시적인 오류가 발생할 수 있습니다. 이러한 상황이 발생하면  경고 표시줄에서 파일 보기를 선택하여 각 파일에 대한 파일 이름, 파일 경로 및 실패 이유가 표시됩니다. 필요한 경우 이 정보를 콤보로 구분된 값 .csv 내보낼 수 있습니다.

콘텐츠에 정보 권한 관리 권한이 포함되어 있는 경우 복사된 콘텐츠에 대해 이러한  사용 권한이 유지 관리되는 것이고, 파일을 열고 보아야 하는 경우 내부자 위험 관리 조사자 역할이 할당된 사용자에게는 이러한 사용 권한 및 권한이 필요합니다. 각 파일 및 메시지에는 관리 목적으로 내부자 위험 관리 사례에서 고유한 파일 ID가 자동으로 할당됩니다. 장치 표시기 활동과 관련된 문서는 콘텐츠 탐색기에서 포함되지 않습니다.

> [!NOTE]
> 콘텐츠 탐색기에는 Microsoft 365, Exchange SharePoint, Microsoft Teams 및 서비스에 대한 사용자 활동과 같은 Microsoft 365 서비스 파일과 관련된 사용자 활동이 비즈니스용 OneDrive.

## <a name="column-options"></a>열 옵션

위험 분석가와 조사자는 캡처된 데이터 및 메시지를 보다 쉽게 검토하고 사례에 대한 컨텍스트를 검토할 수 있도록 콘텐츠 탐색기에 몇 가지 필터링 및 정렬 도구가 포함되어 있습니다. 기본 정렬의 경우 **날짜** 및 파일 클래스 열은 콘텐츠 큐 창의 열 제목을 사용하여 정렬을 지원합니다.  다른 큐 열을 보기에 추가하여 파일 및 메시지에 서로 다른 피벗을 제공할 수 있습니다.

콘텐츠 큐의 열 제목을 추가하거나 제거하려면 열 편집 컨트롤을 **사용하여** 다음 열 옵션에서 선택합니다. 이러한 열은 콘텐츠 탐색기에서 지원되는 일반, 전자 메일 및 문서 속성 조건에 매핑되어 있으며 이 문서의 뒷부분에 나와 있습니다.

| **Bcc** | 전자 메일 메시지, Bcc 메시지 필드의 사용자에 사용할 수 있습니다. |
| **Cc** | 전자 메일 메시지,Cc 메시지 필드의 사용자에 사용할 수 있습니다. |
| **복합 경로** | 항목의 원본을 설명하는 사람이 읽을 수 있는 경로입니다. |
| **대화 ID** | 메시지의 대화 ID입니다. |
| **대화 인덱스** | 메시지의 대화 인덱스입니다. |
| **만든 시간** | 파일 또는 전자 메일 메시지를 만든 시간입니다. |
| **날짜(UTC)** | 전자 메일 메시지의 경우 받는 사람이 메시지를 받았거나 보낸 사람이 메시지를 보낸 날짜입니다. 문서의 경우 문서를 마지막으로 수정한 날짜입니다. 날짜는 UTC(협정 세계시)입니다.|
| **주 테마** | 분석에 대해 계산된 주 테마입니다. |
| **전자 메일 집합 ID** | 동일한 전자 메일 집합의 모든 메시지에 대한 그룹 ID입니다. |
| **가족 ID** | 패밀리 ID는 모든 항목을 함께 그룹화합니다. 전자 메일의 경우 이 열에는 메시지와 모든 첨부 파일이 포함됩니다. 문서의 경우 이 열에는 문서와 포함된 항목이 포함됩니다. |
| **파일 클래스** | 문서 및 SharePoint 콘텐츠의 OneDrive: **문서**; 의 콘텐츠에 Exchange: **전자 메일** 또는 첨부 파일입니다.  |

| **파일 형식 아이콘** | 파일의 확장명입니다. 예를 들어 docx, 1, pptx 또는 xlsx입니다. 이 필드는 FileExtension 사이트 속성과 같은 속성입니다. |
| **ID** | 파일의 GUID 식별자입니다. |
| **Immutable ID** | 2016년 8월 2일부로 저장된 Office 365. |
| **포괄 유형** | 분석에 대해 계산된 포함 유형: **0** - 포괄이 아지 않습니다. **1** - 포괄; **2** - 포괄 마이너스 **3** - 포괄 복사본입니다. |
| **마지막으로 수정한 날짜** | 문서를 마지막으로 변경한 날짜입니다. |

| **메시지 종류** | 검색할 전자 메일 메시지의 유형입니다. 가능한 값: 연락처, docs, 전자 메일, 외부 데이터, 팩스, im, 저널, 모임, Microsoft 팀(Microsoft Teams의 채팅, 모임 및 통화 항목 반환), 메모, 게시물, RSS 피드, 작업, 음성 메일 |

| **Received** | 받는 사람이 전자 메일 메시지를 받은 날짜입니다. 이 필드는 Received 전자 메일 속성과 같은 속성입니다. |
| **받는 사람** | 전자 메일 메시지의 모든 받는 사람 필드 이러한 필드는 To, Cc 및 Bcc입니다. |

| **중요한 정보 유형** | 콘텐츠에 식별된 중요한 정보 유형입니다. |

| **To** | 받는 사람 필드에 있는 전자 메일 메시지의 받는 사람입니다. |

## <a name="filtering"></a>필터링

하나 이상의 필터를 사용하여 검색 범위를 좁히고 보다 구체화된 결과 집합을 반환할 수 있습니다. 필터를 설정하려면 **콘텐츠** 큐의 맨 위에 있는 필터를 선택합니다. 많은 필터에는 필터에서 반환되는 결과 범위를 좁히는 데 도움이 되는 추가 필터링 옵션이 포함되어 있습니다. 예를 들어 *날짜 필터에는* 날짜 필터의 시작  날짜 및 종료 날짜를 구성하는  **컨트롤이** 포함됩니다. 다음 범주에서 하나 이상의 필터 항목을 선택합니다.

### <a name="common-filters"></a>일반 필터

| **Filter** | **설명** |
|:---------------------|:----------------|
| **날짜(UTC)** | 전자 메일 메시지의 경우 받는 사람이 메시지를 받았거나 보낸 사람이 메시지를 보낸 날짜입니다. 문서의 경우 문서를 마지막으로 수정한 날짜입니다. |
| **보낸 사람/만든 이** | 전자 메일의 경우 메시지를 보낸 사람입니다. 문서의 경우 작성자 필드에  인용된 사람이 문서의 Office 합니다. 여러 개의 이름을 쉼표로 구분하여 입력할 수 있습니다. |
| **원본** | 조직에서 문서의 위치입니다. 예를 들어 특정 사이트 SharePoint 있습니다. |
| **제목/제목** | 전자 메일의 경우 메시지 제목 줄의 텍스트입니다. 문서의 경우 문서 제목입니다. 문서의 Title 속성은 문서에 지정된 메타데이터 Microsoft Office 있습니다. 두 개 이상의 제목 이름을 각 제목에 각 별로 구분하여 입력할 수 있습니다. 두 개 이상의 값은 OR 연산자를 사용하여 논리적으로 연결됩니다. |

### <a name="email-filters"></a>전자 메일 필터

| **Filter** | **설명** |
|:---------------------|:----------------|
| **Bcc** | 전자 메일 메시지의 Bcc 필드입니다. |
| **Cc** | 전자 메일 메시지의Cc 필드입니다. |
| **첨부 파일 포함** | 메시지에 첨부 파일이 있는지 여부를 나타냅니다. 값은 true 또는 **false로 나열됩니다.**  |
| **전자 메일 첨부 파일** | 문서가 첨부 파일인 경우 값은 예로 **표시됩니다.** |
| **포함 문서** | 문서가 전자 메일 메시지에 포함된 경우 값은 예로 **표시됩니다.** |
| **인라인 첨부 파일** | 문서가 전자 메일 메시지의 인라인 첨부 파일인 경우 값은 예로 **표시됩니다.** |
| **참가자** | 전자 메일 메시지의 모든 사용자 필드입니다. 이러한 필드는 From, To, Cc 및 Bcc입니다. |
| **Received** | 받는 사람이 전자 메일 메시지를 받은 날짜입니다. |
| **받는 사람 도메인** | 메시지 받는 사람의 모든 도메인 목록입니다. |
| **받는 사람** | 전자 메일 메시지 받는 사람입니다. |
| **보낸 사람** | 메시지 유형에 대한 보낸 사람(보낸 사람) 필드입니다.  형식은 **DisplayName \<SmtpAddress> 입니다.** |
| **보낸 사람 도메인** | 보낸 사람 도메인입니다. |
| **To** | 전자 메일 메시지의 받는 사람 필드입니다. |
| **전자 메일 집합의 고유** | False이면 첨부 파일이 전자 메일 집합에 중복되어 있습니다. |

## <a name="document-filters"></a>문서 필터

| **필터** | **설명** |

| **마지막으로 수정한 날짜(UTC)** | 문서를 마지막으로 변경한 날짜입니다. 날짜와 시간은 UTC(협정 세계시)에 있습니다. |

| **작업 제품** | 문서의 작업 제품 유형입니다. 예를 들어 문서의 주석이나 태그를 예로 들 수 있습니다. |
