# <a name="insider-risk-management-policies"></a>내부자 위험 관리 정책

내부자 위험 관리 정책은 범위 내 사용자와 경고에 대해 구성된 위험 지표 유형을 지정합니다. 조직의 모든 사용자에게 적용되는 정책을 빠르게 만들거나 정책의 관리에 대한 개별 사용자 또는 그룹을 정의할 수 있습니다. 정책은 콘텐츠 우선 순위를 지원하여 여러 또는 특정 Microsoft Teams, SharePoint 사이트, 데이터 민감도 유형 및 데이터 레이블에 대한 정책 조건을 중점적으로 적용합니다. 템플릿을 사용하여 특정 위험 지표를 선택하고 정책 지표에 대한 이벤트 임계값을 사용자 지정하여 위험 점수와 경고의 수준 및 빈도를 효과적으로 사용자 지정할 수 있습니다. 또한 위험 점수 부스터 및 이상한 검색은 중요도가 높거나 더 비정상적인 사용자 활동을 식별하는 데 도움이 됩니다. 정책 창을 사용하면 경고 활동에 정책을 적용할 기간을 정의할 수 있으며 정책이 활성화된 기간을 결정하는 데 사용됩니다.

내부자 [](https://www.youtube.com/watch?v=kudK5ajZTUo) 위험 관리 정책 구성 비디오를 통해 기본 제공 정책 템플릿을 사용하여 만든 정책이 잠재적인 위험에 대한 조치를 신속하게 취할 수 있는 방법에 대한 개요를 확인할 수 있습니다.

## <a name="policy-dashboard"></a>개인 정보 대시보드

정책 **대시보드를** 사용하면 조직의 정책, 정책 상태, 정책에 사용자를 수동으로 추가하고 각 정책과 연결된 경고 상태를 빠르게 확인할 수 있습니다.

- **정책 이름:** 정책 마법사에서 정책에 할당된 이름입니다.
- **상태:** 각 정책의 상태입니다. 정책 경고 및 권장 사항 수 또는 문제 없는 정책의 경우 *Healthy* 상태를 표시됩니다.  정책을 클릭하여 경고 또는 권장 사항에 대한 상태 세부 정보를 확인할 수 있습니다.
- **활성 경고:** 각 정책에 대한 활성 경고 수입니다.
- **확인된 경고:** 지난 365일 동안 정책에서 발생된 총 경고 수입니다.
- **경고에 대해 수행된** 작업: 지난 365일 동안 확인되거나 해지된 총 경고 수입니다.
- **정책 경고 효과:** 확인된 총 경고에 의해 결정된 비율을 경고에 대해 수행된 총 작업으로 나눈 비율입니다(지난 1년 동안 확인되거나 해지된 경고의 합계).

## <a name="policy-recommendations-from-analytics-preview"></a>분석의 정책 권장 사항(미리 보기)

내부자 위험 분석을 사용하면 내부자 위험 정책을 구성하지 않고도 조직에서 잠재적인 내부자 위험을 평가할 수 있습니다. 이 평가는 조직에서 더 높은 사용자 위험의 잠재적인 영역을 식별하고 구성을 고려할 수 있는 내부자 위험 관리 정책의 유형과 범위를 결정하는 데 도움이 될 수 있습니다.

내부자 위험 분석 및 정책 권장 사항에 대한 자세한 내용은 내부자 위험 관리 [설정: 분석(미리 보기)을 참조하세요.](insider-risk-management-settings.md#analytics-preview)

## <a name="policy-templates"></a>정책 템플릿

내부자 위험 관리 템플릿은 정책에서 사용하는 위험 지표 및 위험 점수 모델 유형을 정의하는 미리 정의된 정책 조건입니다. 정책을 생성하기 전에 각 정책에 정책 생성 마법사에서 할당된 템플릿이 있어야 합니다. 내부자 위험 관리는 각 정책 템플릿에 대해 최대 5개의 정책을 지원합니다. 정책 마법사를 사용하여 새 내부자 위험 정책을 만들면 다음 정책 템플릿 중 하나를 선택할 수 있습니다.

### <a name="data-theft-by-departing-users"></a>떠날 사용자에 의해 데이터 도난

사용자가 조직을 떠나는 경우 일반적으로 퇴사하여 데이터 도난과 관련된 특정 위험 지표가 있습니다. 이 정책 템플릿은 위험 점수에 대한 유출 표시기를 사용하며 이 위험 영역의 감지 및 경고에 중점을 습니다. 퇴사한 사용자를 위한 데이터 도용에는 SharePoint Online에서 파일 다운로드, 파일 인쇄, 고용 재배정 및 종료 날짜 근처의 개인 클라우드 메시징 및 저장소 서비스에 데이터 복사가 포함됩니다. Microsoft 365 HR 커넥터 또는 조직의 Azure Active Directory에서 사용자 계정 삭제를 자동으로 모니터링하는 옵션을 사용하여 이 템플릿은 이러한 활동과 관련된 위험 지표와 사용자 고용 상태와의 상관 관계에 대한 점수가 매기기 시작합니다.

>[!IMPORTANT]
>이 템플릿을 사용할 때 조직의 사용자에 대한 해지 및 종료 날짜 정보를 주기적으로 가져오도록 Microsoft 365 HR 커넥터를 구성할 수 있습니다. 조직에 대한 Microsoft 365 HR 커넥터를 구성하는 단계별 지침은 [HR](import-hr-data.md) 커넥터를 사용하여 데이터 가져오기 문서를 참조하세요. HR 커넥터를 사용하지 않는 경우 정책 마법사에서 트리거 이벤트를 구성할 때 Azure AD에서 삭제된 사용자 계정을 선택해야 합니다.

### <a name="general-data-leaks"></a>일반 데이터 누수

데이터를 보호하고 데이터 누출을 방지하는 것은 대부분의 조직에서 특히 사용자, 장치 및 서비스에서 만든 새 데이터가 빠르게 증가하는 데 끊임없는 과제입니다. 사용자는 데이터 누출 관리가 점점 더 복잡해지고 어려워지는 서비스 및 장치에서 정보를 만들고 저장하고 공유할 수 있습니다. 데이터 누출에는 조직 외부의 정보를 실수로 과도하게 공유하거나 악의적인 의도로 데이터 도난을 포함할 수 있습니다. 할당된 DLP(데이터 손실 방지) 정책 또는 기본 제공 트리거 이벤트를 사용하여 이 템플릿은 의심스러운 SharePoint Online 데이터 다운로드, 파일 및 폴더 공유, 파일 인쇄, 개인 클라우드 메시징 및 저장소 서비스에 데이터 복사에 대한 실시간 검색 점수 지정을 시작합니다.

데이터 *누수* 템플릿을 사용할 때 조직의 심각도 높은 경고에 대한 내부자 위험 정책의 표시기를 트리거하는 DLP 정책을 할당할 수 있습니다. DLP 정책 규칙에 의해 심각도 높은 경고가 생성될 때마다 이 템플릿을 사용하여 만든 내부자 위험 정책은 높은 심각도 DLP 경고를 자동으로 검사합니다. 경고에 내부자 위험 정책에 정의된 범위 내 사용자가 포함되어 있는 경우 이 경고는 내부자 위험 정책에 의해 새 경고로 처리되고 내부자 위험 심각도 및 위험 점수가 할당됩니다. 이 정책을 사용하면 사례에 포함된 다른 활동과 컨텍스트에서 이 경고를 평가할 수 있습니다. DLP 정책을 선택하지 않는 경우 기본 제공 트리거 이벤트를 선택해야 합니다.

#### <a name="data-leaks-policy-guidelines"></a>데이터 누수 정책 지침

내부자 위험 관리 정책에 사용할 DLP 정책을 만들거나 수정할 때 다음 지침을 고려하세요.

- DLP 정책에서 규칙을 구성할 때 문제  보고서 설정을 *높음으로* 할당할 때 데이터 유출 이벤트의 우선 순위를 지정하고 선택적으로 선택합니다. 예를 들어 중요한 문서를 알려진 경쟁 업체에  전자 메일로 보내면 높은 경고 수준 유출 이벤트가 발생해야 합니다. 다른 DLP  정책 규칙의  인시던트 보고서 설정에서 높은 수준을 과도하게 할당하면 내부자 위험 관리 경고 워크플로에서 노이즈가 증가하고 데이터 조사자 및 분석가가 이러한 경고를 제대로 평가하기가 더 어려워질 수 있습니다. 예를 들어 높은  경고 수준을 할당하여 DLP 정책의 거부 활동에 액세스하면 실제로 위험한 사용자 동작 및 활동을 평가하기가 더 어려워집니다.
- DLP 및 내부자 위험 관리 정책 모두에서 범위 내 사용자를 이해하고 적절히 구성해야 합니다. 데이터 누출 템플릿을 사용하여 내부자 위험  관리 정책의 범위 내로 정의된 사용자만 심각도 DLP 정책 경고가 처리됩니다. 또한 심각도 DLP 경고에 대한 규칙에서 범위 내로 정의된 사용자만 내부자 위험 관리 정책에서 고려 사항을 검토합니다. DLP와 내부자 위험 정책 둘 다에서 범위 내 사용자를 충돌하는 방식으로 구성하지 않는 것이 중요합니다.

     예를 들어 DLP 정책 규칙의 범위가 영업 팀의 사용자로만 지정되고 데이터  누출 템플릿에서 만든 내부자 위험 정책이 모든 사용자를 범위 내로 정의한 경우 내부자 위험 정책은 실제로 영업 팀의 사용자에 대해 높은 심각도 DLP 경고만 처리합니다. 내부자 위험 정책은 사용자가 이 예의 DLP 규칙에 정의되지 않은 처리를 위해 높은 우선 순위 DLP 알림을 받지 않습니다. 반대로 데이터 누출 템플릿에서 만든 내부자  위험 관리 정책의 범위가 영업 팀의 사용자로만 범위가 지정되어 있으며 할당된 DLP 정책의 범위가 모든 사용자로 지정되면 내부자 위험 정책은 영업 팀 구성원에 대한 높은 심각도 DLP 경고만 처리합니다. 내부자 위험 관리 정책은 영업 팀에 속하지 않은 모든 사용자에 대해 높은 심각도 DLP 경고를 무시합니다.

- 이 내부자 위험 관리 템플릿에 사용되는 DLP 정책의 문제 보고서  규칙 설정이 심각도 수준 높은 경고에 대해 구성되어 있는지 확인  높은 *심각도* 수준은 트리거 이벤트로, 인시던트 보고서 필드가 *낮음* 또는 보통으로  설정된 DLP 정책의 규칙에서 내부자 위험 관리 경고가 생성되지 *않습니다.*

    ![DLP 정책 경고 설정](../media/insider-risk-DLP-policy-high-severity.png)

     >[!NOTE]
     >기본 제공 템플릿을 사용하여 새 DLP 정책을 만들 때 고급 **DLP** 규칙 만들기 또는 사용자  지정 옵션을 선택하여  심각도 수준에 대한 문제 보고서 설정을 구성해야 합니다.

데이터 누수 템플릿에서 만든  각 내부자 위험 관리 정책에는 DLP 정책이 하나만 할당될 수 있습니다. 데이터 누수 템플릿을 사용하는 내부자 위험 정책에 대해 이벤트를 트리거하고 검색하려는 다양한 활동을 결합하는 전용 DLP 정책을 만드는 **것이** 좋습니다.

조직에 [대한 DLP 정책을 구성하기](create-test-tune-dlp-policy.md) 위한 단계별 지침은 DLP 정책 만들기, 테스트 및 조정 문서를 참조하세요.

### <a name="data-leaks-by-priority-users-preview"></a>우선 순위 사용자에 의해 데이터 누수(미리 보기)

조직의 사용자에 대한 데이터 누출을 방지하고 데이터를 보호하는 것은 자신의 위치, 중요한 정보에 대한 액세스 수준 또는 위험 기록에 따라 다를 수 있습니다. 데이터 누출에는 조직 외부의 매우 중요한 정보를 실수로 과도하게 공유하거나 악의적인 의도로 데이터 도난을 포함할 수 있습니다. 할당된 DLP(데이터 손실 방지) 정책을 사용하여 이 템플릿은 의심스러운 활동의 실시간 검색 점수 지정을 시작하여 심각도 수준이 높은 내부자 위험 경고 및 경고의 가능성이 높아집니다. 우선 순위 사용자는 [](insider-risk-management-settings.md#priority-user-groups-preview) 내부자 위험 관리 설정 영역에 구성된 우선 순위 사용자 그룹에 정의됩니다.

일반 데이터 누출 템플릿과 함께, 조직의 심각도 높은 경고에 대한 내부자 위험 정책의 표시기를 트리거하는 DLP 정책을 할당해야 합니다. 이 템플릿을 사용하여 정책을 만들 때 위의 데이터 누출 정책 지침을 따릅니다. 또한 내부자 위험 관리 설정 우선 순위 사용자 그룹에서 만든 우선 순위 사용자 그룹을  >    >   정책에 할당해야 합니다.

### <a name="data-leaks-by-disgruntled-users-preview"></a>불만이 있는 사용자에 의해 데이터 누출(미리 보기)

이 템플릿을 사용하는 경우 조직의 사용자에 대한 성능 개선 알림, 성능 검토 상태 불량 또는 작업 수준 변경 정보를 주기적으로 가져오도록 Microsoft 365 HR 커넥터를 구성해야 합니다. 조직에 대한 Microsoft 365 HR 커넥터를 구성하는 단계별 지침은 [HR](import-hr-data.md) 커넥터를 사용하여 데이터 가져오기 문서를 참조하세요.

## <a name="prioritize-content-in-policies"></a>정책의 콘텐츠 우선 순위 지정

내부자 위험 관리 정책은 콘텐츠가 저장되는 위치 또는 분류 방법에 따라 콘텐츠의 우선 순위를 더 높게 지정할 수 있도록 합니다. 콘텐츠를 우선 순위로 지정하면 모든 관련 활동에 대한 위험 점수가 증가하여 심각도 높은 경고가 생성될 가능성이 높아집니다. 그러나 관련 콘텐츠에 기본 제공 또는 사용자 지정 중요한 정보 유형을 포함하거나 정책에서 우선 순위로 지정하지 않은 경우 일부 활동은 경고를 생성하지 않습니다.

예를 들어 조직에는 기밀 프로젝트 전용 SharePoint 사이트가 있습니다. 이 SharePoint 사이트의 정보에 대한 데이터 누출은 프로젝트를 손상시킬 수 있으며 성공에 큰 영향을 줄 수 있습니다. 데이터 누수 정책에서 이 SharePoint 사이트의 우선 순위를 지정하면 적격 활동에 대한 위험 점수가 자동으로 증가합니다. 이 우선 순위 지정은 이러한 활동이 내부자 위험 경고를 생성할 가능성을 높이고 경고의 심각도 수준을 높입니다.

정책 마법사에서 내부자 위험 관리 정책을 만들 때 다음 우선 순위 중 선택할 수 있습니다.

- **SharePoint 사이트:** 정의된 SharePoint 사이트의 모든 파일 형식과 관련된 모든 활동에 더 높은 위험 점수가 할당됩니다. 
- **중요한 정보 유형:** 중요한 정보 유형을 포함하는 콘텐츠와 관련된 [모든](sensitive-information-type-entity-definitions.md) 활동에 더 높은 위험 점수가 할당됩니다.
- **민감도 레이블:** 특정 민감도 레이블이 [](sensitivity-labels.md) 적용된 콘텐츠와 관련된 모든 활동에 더 높은 위험 점수가 할당됩니다.

## <a name="sequence-detection-preview"></a>시퀀스 검색(미리 보기)

위험한 활동은 격리된 이벤트로 발생하지 않을 수 있습니다. 이러한 위험은 종종 더 큰 이벤트 시퀀스의 일부입니다. 시퀀스는 높은 위험을 제안할 수 있는 둘 이상의 사용자 활동이 순서대로 수행되는 그룹입니다. 이러한 관련 활동을 파악하는 것은 전반적인 위험을 평가하는 데 중요한 부분입니다. 데이터 도용 또는 데이터 누출 정책에 대해 시퀀스 검색을 사용하도록 설정하면 내부자 위험 관리 사례 내의 사용자 활동 탭에 시퀀스 정보 활동의 인사이트가 표시됩니다.  다음 정책 템플릿은 시퀀스 검색을 지원합니다.

- 떠날 사용자에 의해 데이터 도난
- 일반 데이터 누수
- 우선 순위 사용자에 의해 데이터 누수
- 불만이 있는 사용자의 데이터 누수

이러한 내부자 위험 관리 정책은 특정 지표 및 발생 순서를 사용하여 일련의 위험에서 각 단계를 검색할 수 있습니다. 파일 이름은 시퀀스에서 활동을 매핑할 때 사용됩니다. 이러한 위험은 다음과 같은 네 가지 주요 활동 범주로 구성됩니다.

- **컬렉션:** 이러한 범주 신호는 범위 내 정책 사용자의 다운로드 활동에 초점을 맞추고 있습니다. 이 범주의 작업 예로는 SharePoint 사이트에서 파일을 다운로드할 수 있습니다.
- **유출:** 이러한 범주는 범위 내 정책 사용자의 내부 및 외부 원본에 대한 공유 또는 추출 활동에 초점을 맞추는 신호입니다. 이 범주의 활동 예로는 조직의 첨부 파일이 있는 전자 메일을 외부 받는 사람에게 보냅니다.
- **난독 처리:** 이러한 범주 신호는 범위 내 정책 사용자의 위험한 활동 마스킹에 초점을 맞추고 있습니다. 이 범주의 작업 예로는 장치의 파일 이름을 변경합니다.
- **정리:** 이러한 범주는 범위 내 정책 사용자의 지우기 활동에 초점을 맞추고 있습니다. 이 범주의 작업 예는 장치에서 파일을 삭제하는 것입니다.

>[!NOTE]
>시퀀스 검색에서는 내부자 위험 관리에 대한 전역 설정에서 사용하도록 설정된 표시기 및 정책에서 선택된 표시기를 사용합니다. 적절한 표시기를 선택하지 않은 경우 시퀀스 검색이 작동하지 않습니다.

정책에서 구성할 때 각 시퀀스 검색 유형에 대한 개별 임계값 설정을 사용자 지정할 수 있습니다. 이러한 임계값 설정은 시퀀스와 연결된 파일 볼륨에 따라 알림을 조정합니다.

사용자 활동 보기에서 시퀀스 검색 관리에 대한 자세한 **내용은** [Insider risk management cases: User activity을 참조합니다.](insider-risk-management-cases.md#user-activity)

## <a name="cumulative-exfiltration-detection-preview"></a>누적 유출 감지(미리 보기)

내부자 위험 지표는 내부자 위험 정책의 범위 내 사용자에 대해 매일 평가될 때 비정상적인 수준의 위험 활동을 식별하는 데 도움이 됩니다. 누적 유출 감지는 기계 학습 모델을 사용하여 사용자 유출 활동이 시간 및 여러 유출 활동 유형에 따라 측정될 때 조직 평균을 초과하는 경우를 식별하는 데 도움이 됩니다. 내부자 위험 관리 분석가와 조사자는 누적 유출 감지 정보를 사용하여 일반적으로 경고를 생성하지는 않지만 조직에 일반적인 정보보다 높은 유출 활동을 식별할 수 있습니다. 일부 예로는 사용자가 며칠 동안 데이터를 느리게 유출하거나 사용자가 조직의 데이터 공유를 위해 평소보다 더 많은 여러 채널에서 데이터를 반복적으로 공유하는 경우를 예로 들 수 있습니다.

누적 유출 검색은 다음 정책 템플릿을 사용할 때 기본적으로 사용하도록 설정됩니다.

- 떠날 사용자에 의해 데이터 도난
- 일반 데이터 누수
- 우선 순위 사용자에 의해 데이터 누수
- 불만이 있는 사용자의 데이터 누수

>[!NOTE]
>누적 유출 감지는 정책에서 선택된 내부자 위험 관리 및 유출 표시기에 대한 전역 설정에서 사용하도록 설정된 유출 표시기를 사용합니다. 따라서 누적 유출 검색은 선택한 필수 유출 표시기에서만 평가됩니다.

데이터 도용 또는 데이터 누출 정책에 대해 누적 유출 감지를 사용하도록 설정하면 누적 유출 활동의 인사이트가 내부자 위험 관리 사례 내의 사용자 활동 탭에 표시됩니다. 

사용자 활동 관리에 대한 자세한 내용은 내부자 위험 관리 [사례: 사용자 활동을 참조합니다.](insider-risk-management-cases.md#user-activity)

## <a name="policy-health-preview"></a>정책 상태(미리 보기)

정책 상태는 내부자 위험 관리 정책의 잠재적인 문제에 대한 정보를 제공합니다. 정책 탭의 상태 열은 사용자 활동이 보고되지 않도록 할 수 있는 정책 문제나 활동 알림 수가 비정상적인 이유를 경고할 수 있습니다. 또한 정책 상태는 정책이 정상 상태고 주의 또는 구성 변경이 필요하지 않은지 확인할 수 있습니다.

정책에 문제가 있는 경우 정책 상태는 정책 문제를 해결하는 데 도움이 되는 알림 경고 및 권장 사항을 표시합니다. 이러한 알림은 다음 문제를 해결하는 데 도움이 될 수 있습니다.

- 구성이 불완전한 정책 이러한 문제에는 정책에 사용자 또는 그룹이 누락되거나 기타 불완전한 정책 구성 단계가 포함됩니다.
- 표시기 구성 문제가 있는 정책입니다. 표시기는 각 정책에서 중요한 부분입니다. 지표가 구성되지 않은 경우 또는 너무 적게 선택한 지표가 선택된 경우 정책이 예상대로 위험한 활동을 평가하지 않을 수 있습니다.
- 정책 트리거가 작동하지 않는 경우 또는 정책 트리거 요구 사항이 제대로 구성되지 않았습니다. 정책 기능은 다른 서비스 또는 구성 요구 사항에 따라 이벤트 트리거를 효과적으로 감지하여 정책의 사용자에게 위험 점수 할당을 활성화할 수 있습니다. 이러한 종속성에는 커넥터 구성, 끝점 경고 공유용 Microsoft Defender 또는 데이터 손실 방지 정책 구성 설정 관련 문제가 있을 수 있습니다.
- 볼륨 제한이 한도에 도달하거나 초과합니다. 내부자 위험 관리 정책은 다양한 Microsoft 365 서비스 및 끝점을 사용하여 위험 활동 신호를 집계합니다. 정책의 사용자 수에 따라 볼륨 제한으로 인해 위험 활동의 식별 및 보고가 지연될 수 있습니다. 이 문서의 정책 템플릿 제한 섹션에서 이러한 제한에 대해 자세히 알아보십시오.

정책의 상태를 빠르게 확인하기 위해 정책 탭 및 상태 열을 탐색합니다. 각 정책에 대해 다음과 같은 정책 상태 옵션이 표시됩니다.

- 정상: 정책에 문제가 확인되지 않았습니다.
- 권장 사항: 정책에 예상대로 정책이 작동되지 않을 수 있는 몇 가지 문제가 있습니다.
- 경고: 정책에 위험한 활동을 식별하지 못하게 하는 문제가 있습니다.

권장 사항 또는 경고에 대한 자세한 내용을 확인하려면 정책 탭에서 정책을 선택하여 정책 세부 정보 카드를 열 수 있습니다.  이러한 문제를 해결하기 위한 지침을 포함하여 권장 사항 및 경고에 대한 자세한 내용은 세부 정보 카드의 알림 섹션에 표시됩니다.

## <a name="policy-template-limits"></a>정책 템플릿 제한

내부자 위험 관리 정책 템플릿은 제한을 사용하여 범위 내 사용자 위험 활동에 대한 처리량 및 처리 속도와 이 프로세스가 지원 Microsoft 365 서비스와 통합되는 방법을 관리합니다. 각 정책 템플릿에는 정책에 대해 위험 점수를 적극적으로 할당할 수 있는 최대 사용자 수가 있으며, 이러한 사용자는 위험 활동을 지원하고 효과적으로 처리하고 보고할 수 있습니다. 범위 내 사용자는 정책에 대한 이벤트를 트리거하는 사용자입니다.

각 정책의 제한은 정책 템플릿 유형별로 위험 점수를 받는 고유 사용자의 총 수를 기반으로 계산됩니다. 정책 템플릿 유형의 사용자 수가 사용자 제한에 도달하거나 초과하면 정책 성능이 감소합니다. 정책의 현재 사용자 수를 표시하기 위해 정책 탭 및 범위 열의 사용자로 이동합니다. 모든 정책 템플릿에 대해 최대 5개의 정책을 사용할 수 있습니다. 이러한 최대 제한은 특정 정책 템플릿을 사용하는 모든 정책의 사용자에게 적용됩니다.

다음 표를 사용하여 각 정책 템플릿에 대해 지원되는 최대 범위 내 사용자 수를 확인할 수 있습니다.

|**정책 템플릿**|**현재 범위 내 사용자 최대값**|
|:------------------|:--------------------------------|
| 일반 데이터 누수 | 15,000 |
| 불만이 있는 사용자에 의해 데이터 누출 | 7,500 |
| 우선 순위 사용자에 의해 데이터 누출 | 1,000 |
| 떠날 사용자에 의해 데이터 도난 | 20,000 |
| 일반 보안 정책 위반 | 1,000 |
| 우선 순위 사용자에 대한 보안 정책 위반 | 1,000 |
| 사용자를 떠날 경우 보안 정책 위반 | 15,000 |
| 불만이 있는 사용자의 보안 정책 위반 | 7,500 |

## <a name="create-a-new-policy"></a>새 정책 만들기

새 내부자 위험 관리 정책을 만들 수 있도록 Microsoft  365 규정 준수 센터의 내부자 위험 관리 솔루션에서 정책 마법사를 사용하게 됩니다.

새 정책을 만들 수 있는 단계는 다음과 같습니다.

1. Microsoft [365 규정](https://compliance.microsoft.com)준수 센터에서 내부자 위험 관리로 **이동하여** 정책 **탭을** 선택합니다.
2. 정책 **만들기를 선택하여** 정책 마법사를 열 수 있습니다.
3. 정책 **템플릿 페이지에서** 정책 범주를 선택한 다음 새 정책에 대한 템플릿을 선택합니다. 이러한 템플릿은 검색하고 조사하려는 위험 활동을 정의하는 조건 및 표시기로 구성됩니다. 템플릿 선행 요구 사항을 검토하고, 이벤트를 트리거하고, 검색된 활동을 검토하여 이 정책 템플릿이 요구에 맞는지 확인할 수 있습니다.

    >[!IMPORTANT]
    >일부 정책 템플릿에는 관련 경고를 생성하기 위해 정책에 대해 구성해야 하는 선행 구성이 있습니다. 해당 정책 선행 단계를 구성하지 않은 경우 위의 **4단계를** 참조합니다.

4. 계속하려면 **다음을** 선택합니다.
5. 이름 **및 설명 페이지에서** 다음 필드를 입력합니다.
    - **이름(필수)**: 정책의 이름을 입력합니다. 이 이름은 정책을 만든 후에 변경할 수 없습니다.
    - **설명(선택 사항)**: 정책에 대한 설명을 입력합니다.

6. 계속하려면 **다음을** 선택합니다.
7. 사용자 및 **그룹 페이지에서**  모든 사용자 및  그룹 포함 또는 특정 사용자 및 그룹 포함을 선택하여 정책에 포함된 사용자 또는 그룹을 정의하거나 우선 순위 사용자 기반 템플릿을 선택한 경우 우선 **순위 사용자 그룹 추가 또는 편집을 선택합니다.** 모든 사용자 **및 그룹** 포함을 선택하면 조직의 모든 사용자 및 그룹에 대해 정책에 대한 위험 점수 할당을 시작하는 이벤트가 검색됩니다. 특정 사용자 **및 그룹 포함을 선택하면** 정책에 할당할 사용자 및 그룹을 정의할 수 있습니다.
8. 계속하려면 **다음을** 선택합니다.
9. 우선 **순위를** 지정할 콘텐츠 페이지에서 우선 순위를 지정할 원본을 할당(필요한 경우)할 수 있습니다. 이렇게 하면 이러한 원본에 대해 높은 심각도 경고가 생성될 가능성이 높아집니다. 다음 선택 사항 중 하나를 선택합니다.

    - **SharePoint 사이트,** 민감도 레이블 및/또는 중요한 정보 유형을 우선 순위 콘텐츠로 지정하려는 경우 이 옵션을 선택하면 마법사에서 세부 정보 페이지를 사용하여 이러한 채널을 구성할 수 있습니다.
    - 지금 우선 순위 콘텐츠를 지정하지 않습니다(정책을 만든 후 이 작업을 할 **수 있습니다).** 이 옵션을 선택하면 마법사의 채널 세부 정보 페이지가 건너뜁니다.

10. 계속하려면 **다음을** 선택합니다.

11. 이전 단계에서 **SharePoint 사이트,** 민감도 레이블 및/또는 중요한 정보 유형을 우선 순위 콘텐츠로 지정하려는 경우 *SharePoint* *사이트,* 중요한 정보 유형 및 민감도 레이블에 대한 세부 정보 페이지가 표시됩니다.  이러한 세부 정보 페이지를 사용하여 정책의 우선 순위를 지정하기 위해 SharePoint, 중요한 정보 유형 및 민감도 레이블을 정의합니다.

    - **SharePoint 사이트:** **SharePoint** 사이트 추가를 선택하고 액세스 권한이 있으며 우선 순위를 지정할 SharePoint 사이트를 선택합니다. 예를 들어 *"group1@contoso.sharepoint.com/sites/group1" 를 사용할 수 있습니다.*
    - **중요한 정보 유형:** 중요한 정보 **유형 추가를 선택하고** 우선 순위를 지정할 민감도 유형을 선택합니다. 예를 들어 "미국 은행 계좌 *번호"* 및 *"신용 카드 번호"를 예로 들 수 있습니다.*
    - **민감도 레이블:** **민감도** 레이블 추가를 선택하고 우선 순위를 지정할 레이블을 선택합니다. 예를 들어 *"기밀"* *및 "비밀"입니다.*

12. 계속하려면 **다음을** 선택합니다.
13. 지표 **및** 트리거 이벤트 페이지에는 내부자 위험 [](insider-risk-management-settings.md#indicators) 설정 지표 페이지에서 사용할 수 있는 것으로 정의한 지표가  >   표시됩니다. 마법사 시작  시 데이터 누수 템플릿을 선택한 경우 DLP 정책 드롭다운 목록에서 **DLP** 정책을 선택하여 정책에 대한 트리거 표시기를 사용하도록 설정하거나 기본 제공 트리거 이벤트를 선택해야 합니다.

    >[!IMPORTANT]
    >이 페이지의 표시기를 선택할 수 없는 경우 모든 정책에 대해 사용하도록 설정할 표시기를 선택해야 합니다. 마법사에서  표시기 켜기 단추를 사용하거나 내부자 위험 관리 설정 정책 표시기 페이지에서 표시기를 선택할  >    >  **수** 있습니다.

    정책에 적용할 표시기를 선택합니다. 이러한 표시기에서 기본 정책 임계값 설정을 사용하지 않도록 설정하려면 **Microsoft에서** 권장하는 기본 임계값 사용을 사용하지 않도록 설정하고 선택한 각 표시기에서 임계값을 입력합니다.

    - 하나 이상의 *Office* 또는 장치  표시기를 선택한 경우 위험 점수 **부스터를** 적절하게 선택합니다. 위험 점수 부스터는 선택한 지표에만 적용할 수 있습니다.
    - 데이터 도용 또는  데이터 누수 정책 템플릿을 선택한 경우 하나 이상의  **Sequence** 검색 방법과 정책에 적용할 누적 유출 감지 방법을 선택합니다. 

14. 계속하려면 **다음을** 선택합니다.
15. 표시기 **임계값 페이지에서** 기본 표시기 임계값을 사용하거나 개별 지표에 대한 사용자 지정 임계값을 지정하는 옵션을 선택합니다. 각 표시기에서 적절한 수준을 선택해 원하는 수준의 활동 경고를 생성합니다.
16. 계속하려면 **다음을** 선택합니다.
17. 검토 **페이지에서** 정책에 대해 선택한 설정과 선택에 대한 제안 또는 경고를 검토합니다. **편집을** 선택하여 정책 값을 변경하거나  제출을 선택하여 정책을 만들고 활성화합니다.

## <a name="update-a-policy"></a>정책 업데이트

기존 내부자 위험 관리 정책을 업데이트하기 위해 Microsoft  365 규정 준수 센터의 내부자 위험 관리 솔루션에서 정책 마법사를 사용하게 됩니다.

다음 단계를 완료하여 기존 정책을 관리합니다.

1. Microsoft [365 규정](https://compliance.microsoft.com)준수 센터에서 내부자 위험 관리로 **이동하여** 정책 **탭을** 선택합니다.
2. 정책 대시보드에서 관리할 정책을 선택합니다.
3. 정책 세부 정보 페이지에서 정책 **편집을 선택합니다.**
4. 정책 마법사에서는 다음을 편집할 수 없습니다.
    - **정책 템플릿:** 정책에서 모니터링하는 위험 지표 유형을 정의하는 데 사용되는 템플릿입니다.
    - **이름:** 정책의 이름
5. 이름 **및 설명 페이지에서** 설명 필드에서 정책에 대한 **설명을 업데이트합니다.**
6. 계속하려면 **다음을** 선택합니다.
7. 사용자 및 **그룹 페이지에서**  모든 사용자 및  그룹 포함 또는 특정 사용자 및 그룹 포함을 선택하여 정책에 포함된 사용자 또는 그룹을 정의하거나 우선 순위 사용자 기반 템플릿을 선택한 경우 우선 **순위 사용자 그룹 추가 또는 편집을 선택합니다.** 모든 사용자 **및 그룹** 포함을 선택하면 조직의 모든 사용자 및 그룹에 대해 정책에 대한 위험 점수 할당을 시작하는 이벤트가 검색됩니다. 특정 사용자 **및 그룹 포함을 선택하면** 정책에 할당할 사용자 및 그룹을 정의할 수 있습니다.
8. 계속하려면 **다음을** 선택합니다.
9. 우선 **순위를** 지정할 콘텐츠 페이지에서 우선 순위를 지정할 원본을 할당(필요한 경우)할 수 있습니다. 이렇게 하면 이러한 원본에 대해 높은 심각도 경고가 생성될 가능성이 높아집니다. 다음 선택 사항 중 하나를 선택합니다.

    - **SharePoint 사이트,** 민감도 레이블 및/또는 중요한 정보 유형을 우선 순위 콘텐츠로 지정하려는 경우 이 옵션을 선택하면 마법사에서 세부 정보 페이지를 사용하여 이러한 채널을 구성할 수 있습니다.
    - 지금 우선 순위 콘텐츠를 지정하지 않습니다(정책을 만든 후 이 작업을 할 **수 있습니다).** 이 옵션을 선택하면 마법사의 채널 세부 정보 페이지가 건너뜁니다.

10. 계속하려면 **다음을** 선택합니다.

11. 이전 단계에서 **SharePoint 사이트,** 민감도 레이블 및/또는 중요한 정보 유형을 우선 순위 콘텐츠로 지정하려는 경우 *SharePoint* *사이트,* 중요한 정보 유형 및 민감도 레이블에 대한 세부 정보 페이지가 표시됩니다.  이러한 세부 정보 페이지를 사용하여 정책의 우선 순위를 지정하기 위해 SharePoint, 중요한 정보 유형 및 민감도 레이블을 정의합니다.

    - **SharePoint 사이트:** **SharePoint** 사이트 추가를 선택하고 액세스 권한이 있으며 우선 순위를 지정할 SharePoint 사이트를 선택합니다. 예를 들어 *"group1@contoso.sharepoint.com/sites/group1" 를 사용할 수 있습니다.*
    - **중요한 정보 유형:** 중요한 정보 **유형 추가를 선택하고** 우선 순위를 지정할 민감도 유형을 선택합니다. 예를 들어 "미국 은행 계좌 *번호"* 및 *"신용 카드 번호"를 예로 들 수 있습니다.*
    - **민감도 레이블:** **민감도** 레이블 추가를 선택하고 우선 순위를 지정할 레이블을 선택합니다. 예를 들어 *"기밀"* *및 "비밀"입니다.*

12. 계속하려면 **다음을** 선택합니다.
13. 지표 **및** 트리거 이벤트 페이지에는 내부자 위험 [](insider-risk-management-settings.md#indicators) 설정 지표 페이지에서 사용할 수 있는 것으로 정의한 지표가  >   표시됩니다. 마법사 시작  시 데이터 누수 템플릿을 선택한 경우 DLP 정책 드롭다운 목록에서 **DLP** 정책을 선택하여 정책에 대한 트리거 표시기를 사용하도록 설정하거나 기본 제공 트리거 이벤트를 선택해야 합니다.

    >[!IMPORTANT]
    >이 페이지의 표시기를 선택할 수 없는 경우 모든 정책에 대해 사용하도록 설정할 표시기를 선택해야 합니다. 마법사에서  표시기 켜기 단추를 사용하거나 내부자 위험 관리 설정 정책 표시기 페이지에서 표시기를 선택할  >    >  **수** 있습니다.

    정책에 적용할 표시기를 선택합니다. 이러한 표시기에서 기본 정책 임계값 설정을 사용하지 않도록 설정하려면 **Microsoft에서** 권장하는 기본 임계값 사용을 사용하지 않도록 설정하고 선택한 각 표시기에서 임계값을 입력합니다.

    - 하나 이상의 *Office* 또는 장치  표시기를 선택한 경우 위험 점수 **부스터를** 적절하게 선택합니다. 위험 점수 부스터는 선택한 지표에만 적용할 수 있습니다.
    - 데이터 도용 또는  데이터 누수 정책 템플릿을 선택한 경우 하나 이상의  **Sequence** 검색 방법과 정책에 적용할 누적 유출 감지 방법을 선택합니다. 

14. 계속하려면 **다음을** 선택합니다.
15. 표시기 **임계값 페이지에서** 기본 표시기 임계값을 사용하거나 개별 지표에 대한 사용자 지정 임계값을 지정하는 옵션을 선택합니다. 각 표시기에서 적절한 수준을 선택해 원하는 수준의 활동 경고를 생성합니다.
16. 계속하려면 **다음을** 선택합니다.
17. 검토 **페이지에서** 정책에 대해 선택한 설정과 선택에 대한 제안 또는 경고를 검토합니다. **편집을** 선택하여 정책 값을 변경하거나  제출을 선택하여 정책을 만들고 활성화합니다.

## <a name="copy-a-policy"></a>정책 복사

기존 정책과 유사하지만 몇 가지 구성 변경만 필요한 새 정책을 만들어야 할 수 있습니다. 새 정책을 처음부터 새로 만드는 대신 기존 정책을 복사한 다음 새 정책에서 업데이트해야 하는 영역을 수정할 수 있습니다.

다음 단계를 완료하여 기존 정책을 복사합니다.

1. Microsoft 365 규정 준수 센터에서 내부자 위험 관리로 이동하여 정책 탭을 선택합니다.
2. 정책 대시보드에서 복사할 정책을 선택합니다.
3. 정책 세부 정보 페이지에서 복사를 선택합니다.
4. 정책 마법사에서 새 정책의 이름을 지정하고 필요한 경우 정책 구성을 업데이트합니다.

## <a name="immediately-start-scoring-user-activity"></a>즉시 사용자 활동 점수 매기기 시작

내부자 위험 관리 트리거 이벤트 워크플로 외부에서 내부자 위험 정책이 있는 사용자에게 즉시 위험 점수를 할당해야 하는 시나리오가 있을 수 있습니다. 정책  탭의 사용자에 대해  점수 매기기 작업을 사용하여 특정 기간 동안 하나 이상의 내부자 위험 정책에 사용자를 수동으로 추가하고, 자신의 작업에 위험 점수를 즉시 할당하기 시작하고, 사용자에게 트리거 표시기(예: DLP 정책 일치)를 가지기 위한 요구 사항을 무시합니다. 사용자의 활동 타임라인에 나타나는 정책에 사용자를 추가하는 이유를 추가할 수 있습니다. 정책에 수동으로 추가된 사용자는  사용자 대시보드에 표시되고 활동이 정책 경고 임계값을 충족하는 경우 알림이 만들어집니다.

사용자 작업의 점수가 즉시 매기기 시작할 수 있는 몇 가지 시나리오는 다음과 같습니다.

- 사용자가 위험 우려로 식별된 경우 하나 이상의 정책에 대한 활동에 위험 점수 할당을 즉시 시작하려는 경우
- 하나 이상의 정책에 대한 관련 사용자의 활동에 위험 점수를 즉시 할당해야 할 수 있는 인시던트가 있는 경우
- HR 커넥터를 아직 구성하지 않았지만 사용자를 위해 .csv 파일을 업로드하여 HR 이벤트에 대한 사용자 활동에 위험 점수 할당을 시작하려는 경우

>[!NOTE]
>수동으로 추가한 새 사용자가 사용자 대시보드에 표시될 때 몇 시간이 걸릴 **수** 있습니다. 이러한 사용자의 지난 90일 동안의 활동은 표시하는 데 최대 24시간이 걸릴 수 있습니다. 수동으로 추가한 사용자에 대한 활동을 확인하려면 사용자 탭으로  이동한 다음  사용자 대시보드에서 사용자를 선택하고 세부 정보 창에서 사용자 활동 탭을 열 수 있습니다. 

하나 이상의 내부자 위험 관리 정책에서 사용자에 대한 점수 매기기 작업을 수동으로 시작하려면 다음 단계를 완료합니다.

1. Microsoft [365 규정](https://compliance.microsoft.com)준수 센터에서 내부자 위험 관리로 **이동하여** 정책 **탭을** 선택합니다.
2. 정책 대시보드에서 사용자를 추가할 정책 또는 정책을 선택합니다.
3. 사용자에 **대한 점수 매기기 활동 시작을 선택합니다.**
4. 여러 **정책에** 사용자  추가 창의 이유 필드에서 사용자를 추가하는 이유를 추가합니다.
5. This **should last for (between 5 and 30 days)** field, define the number of days to score the user's activity for the policy they are added to the policy
6. Active Directory에서 사용자를 검색하기 위해 검색 사용자를 사용하여 정책 **필드에 추가합니다.** 정책에 추가할 사용자의 이름을 입력합니다. 사용자 이름을 선택하고 반복하여 정책에 추가 사용자를 할당합니다. 선택한 사용자 목록은 여러 정책에 사용자 추가 창의 사용자 섹션에 표시됩니다.
7. 정책에 추가할 사용자 목록을 가져오기 위해  가져오기 를 선택하여 .csv(콤보로 구분된 값) 파일을 가져올 수 있습니다. 파일은 다음 형식의 파일에서 사용자 계정 이름을 나열해야 합니다.

    ```csv
    user principal name
    user1@domain.com
    user2@domain.com
    ```

8. 정책에 사용자 추가를 선택하여 변경 내용을 수락하고 정책에 사용자를 추가하거나 취소를 선택하여 변경 내용을 취소하고 대화 상자를 닫습니다.

## <a name="stop-scoring-users-in-a-policy"></a>정책에서 사용자 점수 매기기 중지

정책에서 사용자 점수 지정을 중지하려면 [Insider Risk Management Users: Remove users from in-scope assignment to policies 문서를 참조하세요.](insider-risk-management-users.md#remove-users-from-in-scope-assignment-to-policies)

## <a name="delete-a-policy"></a>정책 삭제

>[!NOTE]
>정책을 삭제하면 정책에서 생성된 활성 또는 보관된 경고는 삭제되지 않습니다.

기존 내부자 위험 관리 정책을 삭제하려면 다음 단계를 완료합니다.

1. Microsoft [365 규정](https://compliance.microsoft.com)준수 센터에서 내부자 위험 관리로 **이동하여** 정책 **탭을** 선택합니다.
2. 정책 대시보드에서 삭제할 정책을 선택합니다.
3. 대시보드 **도구 모음에서** 삭제를 선택합니다.
4. 삭제 **대화 상자에서** 예를 **선택하여** 정책을 삭제하거나 취소를 **선택하여** 대화 상자를 닫습니다.
