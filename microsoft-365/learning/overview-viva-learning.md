# <a name="overview-of-microsoft-viva-learning"></a>개요 Microsoft Viva Learning

Viva Learning 사용하여 학습 및 구축 기술을 Microsoft Teams 통합할 수 있는 중앙 집중식 학습 허브입니다. Viva Learning 팀에서는 조직과 파트너가 제공하는 콘텐츠 라이브러리를 검색, 공유, 권장 및 학습할 수 있습니다. 모든 작업을 종료하지 않고도 이 작업을 Microsoft Teams.

   ![Screenshot of the Viva Learning homepage in Teams.](../media/learning/overview-1.png)

Viva Learning 사용하면 이미 사용 중이던 커뮤니케이션 도구를 멀어지지 않고도 조직에 대한 학습 및 성장 기회를 쉽게 만들 수 있습니다.

## <a name="learn-while-working"></a>작업 중 학습

### <a name="everyone"></a>모든 사용자

Viva Learning 쉽게 학습을 통합할 수 있습니다. Viva Learning 열면 Microsoft Teams LinkedIn과 같은 조직 및 파트너의 학습 콘텐츠에 대한 개인 설정 보기가 Learning. 더 많은 교육을 계속 검색하고 완료하면 관심사에 반영하기 위해 권장 콘텐츠가 업데이트됩니다.

- 조직에서 제공하는 학습 기회를 쉽게 찾을 수 있습니다.
- Microsoft 및 타사 콘텐츠 공급자의 과정을 검색합니다.
- 도움이 되거나 경력 목표를 지원하는 특정 학습 콘텐츠를 검색합니다.
- 채팅 또는 채널에서 팀 구성원 또는 그룹과 관련, 흥미롭고 중요한 학습 Microsoft Teams 공유합니다.
- 채널 및 탭에서 학습 콘텐츠의 사용자 지정 Microsoft Teams 구성합니다.
- 관심 있는 책갈피 과정입니다.
- LinkedIn Learning 종료하지 않고 포함된 플레이어의 과정을 Microsoft Teams.

### <a name="managers"></a>관리자

플랫폼 전체에서 학습을 조정하지 않고도 팀이 필요한 기술에 참여하고 최신 정보를 유지하세요. 개인에게 콘텐츠를 학습하고, 팀과 콘텐츠를 공유하고, 권장한 학습의 보고된 완료 상태를 추적할 수 있습니다.

## <a name="admin-roles"></a>관리자 역할

Viva Learning 일부 콘텐츠는 기본적으로 Microsoft Teams 사용할 수 있습니다. Viva 2013에서 학습 콘텐츠 Learning 설정하고 개별 라이선스를 관리하려면 다음 권한이 필요합니다.

- Microsoft Teams 관리자
- Microsoft 365 관리자 또는 SharePoint 관리자
- 지식 관리자

### <a name="knowledge-admin"></a>지식 관리자

지식 관리자는 조직의 모든 Azure Active Directory 할당할 수 있는 Microsoft 365 관리 센터(Azure AD) 역할입니다. 이 역할은 조직의 학습 콘텐츠 원본을 관리하기 위해 Microsoft 365 관리 센터. 자세한 내용은 [Azure AD 기본 제공 역할](/azure/active-directory/roles/permissions-reference#knowledge-administrator)을 참조하세요.

지식 관리자는 보통 기술적인 지식이 있으며 관리자 자격 증명을 SharePoint 합니다. 지식 관리자는 조직의 교육, 학습, 교육 또는 직원 경험에서 잘 교육을 쌓아야 합니다.

## <a name="learning-content-sources"></a>Learning 원본

Microsoft Learn and Microsoft 365 Training의 콘텐츠는 Viva Learning. LinkedIn 2013에서 125개 과정 집합에 무료로 액세스할 Learning. 또한 Viva Learning 타사 콘텐츠 공급자 및 학습 관리 시스템과 통합할 수 있습니다. 콘텐츠 원본을 설정하는 방법에 대한 자세한 내용은 [Manage content sources for Viva Learning.](content-sources-365-admin-center.md)

## <a name="data-and-privacy"></a>데이터 및 개인 정보

Viva Learning 데이터 상주는 테넌트 관련이 있으며 사용 가능한 Microsoft 365 표준 데이터 저장소 지침을 따릅니다. 자세한 내용은 [Where is my Microsoft 365 customer data 를 참조하십시오.](/microsoft-365/enterprise/o365-data-locations)

통합 SharePoint 현재 테넌트의 홈 지리에서 호스팅되는 사이트에만 지원됩니다. 예를 들어 프랑스 테넌트는 프랑스에서 호스팅되는 SharePoint Viva 사이트에만 연결될 수 Learning.

Viva 2013에서 저장되는 Learning 포함:

- Learning, 설명, 만든 이 및 언어와 같은 개체 콘텐츠 메타데이터
- 책갈피, 최근에 본 과정, 권장 과정, 할당된 과정 및 완료 레코드와 같은 사용자 데이터
- 필요한 서비스 데이터(예: 오류 로그)
- 진단 데이터 사용

>[!NOTE]
>관리자는 진단 데이터의 저장소를 설정하거나 해제할 수 있습니다.

## <a name="get-started"></a>시작하기

사용자 환경에서 Viva 2016을 설정하고 구성할 준비가 Learning 다음을 Microsoft 365 있습니다.

- Microsoft Teams 관리 센터를 사용하여 조직 [전체에서 Viva Learning 관리합니다.](set-up-viva-learning.md)
- 다음 Microsoft 365 관리 센터 사용하여 특정 그룹에서 사용할 수 있는 학습 [원본을 구성합니다.](content-sources-365-admin-center.md)
- SharePoint 관리 센터를 사용하여 학습 콘텐츠를 관리하고 [저장합니다.](configure-sharepoint-content-source.md)
